# 泛型

## 概述

泛型（Generics）是 Java 5 引入的特性，支持在编译时进行类型安全检查，避免强制类型转换。

核心优势：
- **类型安全**：编译时检查，减少 `ClassCastException`
- **代码复用**：编写可适用于多种类型的通用代码
- **消除强制转换**：代码更简洁清晰

```java
// 没有泛型：需要强制转换
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);  // 需要强制转换

// 使用泛型：类型安全
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);  // 无需强制转换
```

## 类型参数命名约定

- `<E>` - Element（集合元素）
- `<K, V>` - Key, Value（键值对）
- `<T>` - Type（一般类型）
- `<N>` - Number（数字类型）
- `<R>` - Result（返回类型）
- `<S, U, V>` - 多类型参数

## 泛型类

```java
// 普通泛型类
public class Box<T> {
    private T content;

    public Box(T content) {
        this.content = content;
    }

    public T getContent() {
        return content;
    }

    public void setContent(T content) {
        this.content = content;
    }
}

// 使用
Box<String> stringBox = new Box<>("Hello");
String content = stringBox.getContent();

Box<Integer> intBox = new Box<>(42);
Integer num = intBox.getContent();
```

## 泛型方法

```java
public class ArrayUtil {

    // 泛型方法
    public static <T> T findFirst(T[] array, Predicate<T> predicate) {
        for (T element : array) {
            if (predicate.test(element)) {
                return element;
            }
        }
        return null;
    }

    // 泛型方法（带边界）
    public static <T extends Comparable<T>> T findMax(T[] array) {
        if (array == null || array.length == 0) {
            return null;
        }
        T max = array[0];
        for (int i = 1; i < array.length; i++) {
            if (array[i].compareTo(max) > 0) {
                max = array[i];
            }
        }
        return max;
    }
}

// 使用
String[] names = {"Alice", "Bob", "Charlie"};
String first = ArrayUtil.findFirst(names, n -> n.startsWith("A"));

Integer[] numbers = {3, 1, 4, 1, 5, 9};
Integer max = ArrayUtil.findMax(numbers);
```

## 有界类型参数

### 上界限定

```java
// T 必须继承或实现 Number
public class NumericBox<T extends Number> {
    private T value;

    public T getValue() {
        return value;
    }

    // 可调用 Number 的方法
    public double toDouble() {
        return value.doubleValue();
    }
}

NumericBox<Integer> intBox = new NumericBox<>(42);
double val = intBox.toDouble();

// 多个上界（用 & 分隔）
public <T extends Comparable<T> & Serializable> void process(T item) {
    // T 必须实现 Comparable 和 Serializable
}
```

### 下界限定

```java
// ? super T 表示 T 或 T 的父类型
public class Container<T> {
    public void addNumbers(List<? super Integer> list) {
        list.add(1);     // 可以添加 Integer
        list.add(2);     // 可以添加 Integer
        // list.add(3.14);  // 编译错误

        // 可以读取，但返回类型是 Object
        Object obj = list.get(0);
    }
}

List<Number> numList = new ArrayList<>();
Container<Number> container = new Container<>();
container.addNumbers(numList);  // 合法：Number 是 Integer 的父类型
```

### 通配符对比

| 符号 | 含义 | 用途 |
|------|------|------|
| `<?>` | 无界通配符 | 只读操作，不知道具体类型 |
| `<? extends T>` | 上界限定 | 读取安全（返回 T） |
| `<? super T>` | 下界限定 | 写入安全（接受 T） |

```java
// PECS: Producer Extends, Consumer Super

// 生产者（读取）：用 extends
public double sumOfList(List<? extends Number> list) {
    double sum = 0;
    for (Number n : list) {  // 可以读取
        sum += n.doubleValue();
    }
    return sum;
}

// 消费者（写入）：用 super
public void addNumbers(List<? super Integer> list) {
    list.add(1);    // 可以写入 Integer
    list.add(2);    // 可以写入 Integer
    // Number n = list.get(0);  // 读取不安全，返回 Object
}

// 既读又写：不用通配符
public void copy(List<? super Integer> dest, List<? extends Integer> src) {
    for (Integer item : src) {  // 从 src 读取
        dest.add(item);          // 写入 dest
    }
}
```

## 类型擦除

Java 泛型通过类型擦除实现：编译时进行类型检查，运行时移除泛型信息。

```java
// 源代码
public class Box<T> {
    private T content;
    public T getContent() { return content; }
}

// 编译后（伪代码）
public class Box {
    private Object content;  // T 被替换为 Object
    public Object getContent() { return content; }
}

// 带边界时
public class NumericBox<T extends Number> {
    private T content;
    public T getContent() { return content; }
}

// 编译后
public class NumericBox {
    private Number content;  // T 被替换为上界 Number
    public Number getContent() { return content; }
}
```

### 类型擦除的影响

```java
// 不能创建泛型数组
public class Stack<E> {
    // E[] array = new E[10];  // 编译错误
    E[] array = (E[]) new Object[10];  // 需要强制转换

    // 不能使用基本类型
    // Stack<int> intStack;  // 编译错误，必须用包装类
    Stack<Integer> intStack;  // 正确
}

// 不能 instanceof 泛型
if (obj instanceof List<String>) {  // 编译错误
    // ...
}
if (obj instanceof List<?>) {  // 正确
    // ...
}

// 不能重载（类型擦除后方法签名相同）
public class Example {
    // public void process(List<String> list) { }  // 编译错误
    // public void process(List<Integer> list) { } // 与上者类型擦除后相同

    public void process(List<String> list) { }
    public void process(List<?> list) { }  // 正确：不同签名
}
```

## 泛型与继承

```java
// Integer 是 Number 的子类型
Integer intObj = 1;
Number numObj = intObj;  // 协变

// 但 List<Integer> 不是 List<Number> 的子类型
List<Integer> intList = new ArrayList<>();
// List<Number> numList = intList;  // 编译错误

// 使用通配符实现协变
List<? extends Number> numList = new ArrayList<Integer>();  // 合法

// 逆变
List<Number> numList2 = new ArrayList<>();
List<? super Integer> intList2 = numList2;  // 合法：Number 是 Integer 的父类型
```

## 泛型擦除与桥方法

```java
public class Node<T> {
    private T data;

    public Node(T data) {
        this.data = data;
    }

    public T getData() {
        return data;
    }

    public void setData(T data) {
        this.data = data;
    }
}

public class StringNode extends Node<String> {
    public StringNode(String data) {
        super(data);
    }

    @Override
    public String getData() {
        return super.getData();
    }

    // 编译器生成桥方法
    @Override
    public Object getData() {  // 桥方法
        return this.getData();  // 调用上面的方法
    }
}
```

## 泛型约束

```java
// 不能实例化泛型类型
public <T> T createInstance(Class<T> clazz) {
    return clazz.newInstance();  // 正确
    // return new T();  // 编译错误
}

// 静态方法不能使用类泛型参数
public class GenericClass<T> {
    // private static T staticField;  // 编译错误

    // 静态泛型方法是允许的
    public static <T> T createInstance() {
        try {
            return (T) Class.forName("some.Class").newInstance();
        } catch (Exception e) {
            return null;
        }
    }
}

// 不能使用泛型创建异常
public class GenericException<T> extends Exception {  // 编译错误
    // ...
}
```

## 泛型最佳实践

### 1. 优先使用泛型集合

```java
// 错误
List list = new ArrayList();
list.add("string");
list.add(123);  // 运行时可能出错

// 正确
List<String> list = new ArrayList<>();
list.add("string");
list.add("123");
```

### 2. 优先使用泛型方法

```java
// 错误：重载方法使用不同泛型签名
public String convert(Integer value) { return value.toString(); }
public String convert(String value) { return value; }

// 正确：使用泛型方法
public <T> String convert(T value) {
    return Objects.toString(value);
}
```

### 3. 使用泛型字面量

```java
// 创建泛型实例
List<String> list = new ArrayList<>();
Map<String, Integer> map = new HashMap<>();

// Diamond 运算符（Java 7+）
List<String> list = new ArrayList<>();  // 省略类型参数

// 使用 var（Java 10+）
var list = new ArrayList<String>();  // 推断为 ArrayList<String>
```

### 4. 泛型和数组

```java
// 泛型数组不能直接创建
// String[] arr = new String[] <String>();  // 编译错误

// 使用 varargs 配合泛型
@SafeVarargs
public static <T> void printAll(T... elements) {
    for (T element : elements) {
        System.out.println(element);
    }
}
```

### 5. 合理使用边界

```java
// 无界通配符：只读操作
public boolean isEmpty(List<?> list) {
    return list.isEmpty();
}

// 上界：读取操作
public double sum(List<? extends Number> list) {
    double sum = 0;
    for (Number n : list) {
        sum += n.doubleValue();
    }
    return sum;
}

// 下界：写入操作
public void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}
```

### 6. 泛型和可变参数

```java
// 使用 @SafeVarargs 避免警告
@SafeVarargs
public static <T> List<T> listOf(T... elements) {
    return Arrays.asList(elements);
}

// 正确使用
List<String> names = listOf("Alice", "Bob", "Charlie");
```

## 泛型与反射

```java
public class GenericFactory<T> {
    private final Class<T> clazz;

    public GenericFactory(Class<T> clazz) {
        this.clazz = clazz;
    }

    public T create() throws Exception {
        return clazz.getDeclaredConstructor().newInstance();
    }

    public T create(Class<?>[] paramTypes, Object... args) throws Exception {
        return clazz.getDeclaredConstructor(paramTypes)
                    .newInstance(args);
    }
}

// 使用
GenericFactory<User> factory = new GenericFactory<>(User.class);
User user = factory.create();
```
