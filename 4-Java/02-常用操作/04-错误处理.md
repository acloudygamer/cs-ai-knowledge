# 错误处理

## 异常体系

Java 所有异常继承自 `Throwable`，分为两大分支：

```
Throwable
├── Error（系统级错误，不应捕获）
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   ├── NoClassDefFoundError
│   └── VirtualMachineError
└── Exception（可处理异常）
    ├── RuntimeException（非受检异常）
    │   ├── NullPointerException
    │   ├── IllegalArgumentException
    │   ├── IndexOutOfBoundsException
    │   ├── ClassCastException
    │   └── ArithmeticException
    └── 其他受检异常
        ├── IOException
        ├── SQLException
        ├── ClassNotFoundException
        └── ReflectiveOperationException
```

### 受检异常 vs 非受检异常

- **受检异常（Checked Exception）**：编译器强制要求处理或声明，调用者必须 catch 或 throws
- **非受检异常（Unchecked Exception）**：继承自 `RuntimeException`，不强制处理

```java
// 受检异常必须声明或捕获
public void readFile() throws IOException {
    FileReader reader = new FileReader("file.txt");
    // ...
}

// 非受检异常可不处理
public void divide(int a, int b) {
    if (b == 0) {
        throw new ArithmeticException("除数不能为零");
    }
    System.out.println(a / b);
}
```

## try/catch/finally

### 基本结构

```java
try {
    // 可能抛出异常的代码
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // 处理特定异常
    System.err.println("算术错误: " + e.getMessage());
} catch (Exception e) {
    // 处理其他异常（子异常在父异常之前捕获）
    System.err.println("未知错误: " + e.getMessage());
} finally {
    // 总是执行，用于资源清理
    System.out.println("清理资源");
}
```

### 多异常捕获（Java 7+）

```java
// 方式1：多个 catch 块
try {
    // 可能抛出 IOException 或 SQLException
} catch (IOException e) {
    handleIOException(e);
} catch (SQLException e) {
    handleSQLException(e);
}

// 方式2：多异常合并（Java 7+）
try {
    // ...
} catch (IOException | SQLException e) {
    // IOException 和 SQLException 共用处理逻辑
    System.err.println("IO 或数据库错误: " + e.getMessage());
}

// 方式3：多异常合并为最终异常（Java 7+，异常为 effectively final）
try {
    // ...
} catch (IOException | SQLException e) {
    // e 是 effectively final
    log(e.getMessage());
    throw e;
}
```

### try-with-resources（Java 7+）

自动关闭实现 `AutoCloseable` 的资源，无需 finally 块。

```java
// 基本用法
try (BufferedReader reader = new BufferedReader(new FileReader("file.txt"))) {
    String line = reader.readLine();
    System.out.println(line);
} catch (IOException e) {
    System.err.println("IO 错误: " + e.getMessage());
}

// 多个资源
try (
    FileInputStream fis = new FileInputStream("input.txt");
    FileOutputStream fos = new FileOutputStream("output.txt")
) {
    // 使用资源
} catch (IOException e) {
    e.printStackTrace();
}

// 自定义资源
public class DatabaseConnection implements AutoCloseable {
    public void query(String sql) { /* ... */ }

    @Override
    public void close() {
        System.out.println("关闭数据库连接");
    }
}

try (DatabaseConnection db = new DatabaseConnection()) {
    db.query("SELECT * FROM users");
}
```

## 异常抛出与自定义

### throw 与 throws

```java
// throw：抛出异常对象
public void validate(int age) {
    if (age < 0) {
        throw new IllegalArgumentException("年龄不能为负数: " + age);
    }
}

// throws：声明方法可能抛出的异常
public void readFile() throws IOException, SQLException {
    // ...
}
```

### 自定义异常

```java
// 业务异常（通常为非受检）
public class BusinessException extends RuntimeException {
    private final String code;

    public BusinessException(String message) {
        super(message);
        this.code = "BUSINESS_ERROR";
    }

    public BusinessException(String code, String message) {
        super(message);
        this.code = code;
    }

    public BusinessException(String message, Throwable cause) {
        super(message, cause);
        this.code = "BUSINESS_ERROR";
    }

    public String getCode() {
        return code;
    }
}

// 包装异常（保留原始异常信息）
public class ServiceException extends Exception {
    private final int errorCode;

    public ServiceException(String message, int errorCode) {
        super(message);
        this.errorCode = errorCode;
    }

    public ServiceException(String message, Throwable cause) {
        super(message, cause);
        this.errorCode = -1;
    }

    public int getErrorCode() {
        return errorCode;
    }
}
```

### 异常链与原因

```java
// 方法1：使用构造函数传递 cause
try {
    // 原始操作
} catch (IOException e) {
    throw new ServiceException("服务失败", e);
}

// 方法2：使用 initCause（Java 6 及以前）
try {
    // 原始操作
} catch (IOException e) {
    ServiceException se = new ServiceException("服务失败");
    se.initCause(e);
    throw se;
}

// 方法3：使用 enhanceNoSuppressed（Java 7+）
try {
    // ...
} catch (IOException e) {
    ServiceException se = new ServiceException("服务失败");
    se.addSuppressed(e);  // 保留被抑制的异常
    throw se;
}

// 获取原因链
Throwable cause = e.getCause();
Throwable[] suppressed = e.getSuppressed();  // 获取被抑制的异常
```

## 异常处理最佳实践

### 1. 只捕获真正能处理的异常

```java
// 错误：捕获过于宽泛
try {
    // ...
} catch (Exception e) {
    e.printStackTrace();  // 错误：吞掉了异常
}

// 正确：捕获具体异常
try {
    // ...
} catch (IOException e) {
    log.error("IO 错误", e);
    throw new ServiceException("无法读取文件", e);
}
```

### 2. 不要用异常控制业务逻辑

```java
// 错误：使用异常做条件判断
try {
    user = userRepository.findById(id);
    if (user == null) {
        throw new UserNotFoundException(id);
    }
} catch (UserNotFoundException e) {
    // 处理找不到用户
}

// 正确：使用 Optional 或返回值
Optional<User> user = userRepository.findById(id);
if (user.isPresent()) {
    // 处理用户
} else {
    // 处理找不到用户
}

// 或使用 null 对象模式
User user = userRepository.findById(id).orElse(User.UNKNOWN);
```

### 3. 清理资源的三种方式

```java
// 方式1：try-with-resources（推荐）
try (Connection conn = dataSource.getConnection()) {
    // 使用连接
}

// 方式2：finally 块
Connection conn = null;
try {
    conn = dataSource.getConnection();
    // 使用连接
} finally {
    if (conn != null) {
        try {
            conn.close();
        } catch (SQLException e) {
            log.error("关闭连接失败", e);
        }
    }
}

// 方式3：null 检查 + finally（较少使用）
```

### 4. 异常日志记录规范

```java
// 记录异常并重新抛出（保留堆栈）
try {
    // ...
} catch (IOException e) {
    log.error("读取文件失败: {}", filePath, e);
    throw new ServiceException("无法读取配置", e);
}

// 不重复记录日志
try {
    // ...
} catch (IOException e) {
    // 不要在 catch 中记录后又重新抛出（会导致重复日志）
    // 除非是包装后转换为业务异常
    throw new ServiceException("读取失败", e);
}

// 确保 finally 中的日志不抛出异常
try {
    // ...
} finally {
    try {
        closeResource();
    } catch (IOException e) {
        log.warn("关闭资源失败", e);  // 只记录，不抛出
    }
}
```

### 5. 异常转换模式

```java
// DAO 层：捕获具体异常，转换为业务异常
public User findById(Long id) {
    try {
        return jdbcTemplate.queryForObject(
            "SELECT * FROM users WHERE id = ?",
            userRowMapper,
            id
        );
    } catch (EmptyResultDataAccessException e) {
        return null;  // 找不到返回 null 是合理的
    } catch (DataAccessException e) {
        throw new DataAccessException("查询用户失败", e);
    }
}

// 服务层：统一异常处理
public User getUser(Long id) {
    User user = userDao.findById(id);
    if (user == null) {
        throw new UserNotFoundException("用户不存在: " + id);
    }
    return user;
}
```

### 6. 异常屏蔽（Suppressed Exceptions）

当 try-with-resources 中资源关闭时抛出异常，会屏蔽原始异常。

```java
try (Connection conn = dataSource.getConnection()) {
    // 业务操作抛出异常
    throw new RuntimeException("业务异常");
} catch (RuntimeException e) {
    // 关闭连接时抛出的异常被抑制
    Throwable[] suppressed = e.getSuppressed();
    for (Throwable t : suppressed) {
        System.err.println("被抑制的异常: " + t.getMessage());
    }
    throw e;  // 重新抛出原始异常
}
```

## 常见异常模式

### 1. 空指针防御

```java
// 错误：嵌套 null 检查
if (user != null) {
    if (user.getAddress() != null) {
        if (user.getAddress().getCity() != null) {
            return user.getAddress().getCity();
        }
    }
}

// 正确：使用 Optional
return Optional.ofNullable(user)
    .map(User::getAddress)
    .map(Address::getCity)
    .orElse("未知");

// 正确：使用 Objects.requireNonNull
public void setName(String name) {
    this.name = Objects.requireNonNull(name, "名称不能为空");
}
```

### 2. 参数验证

```java
public class OrderService {

    public void createOrder(String userId, List<Item> items) {
        // 验证参数
        Objects.requireNonNull(userId, "用户ID不能为空");
        Objects.requireNonNull(items, "订单项不能为空");

        // 验证业务规则
        if (items.isEmpty()) {
            throw new IllegalArgumentException("订单项不能为空");
        }

        for (Item item : items) {
            if (item.getQuantity() <= 0) {
                throw new IllegalArgumentException("商品数量必须大于0");
            }
        }
    }

    // 使用 Jakarta Bean Validation
    public void createOrder(@NotNull String userId, @NotEmpty List<@NotNull Item> items) {
        // ...
    }
}
```

### 3. 资源安全模式

```java
public String readConfig(String path) {
    // 验证路径
    if (path == null || path.isEmpty()) {
        throw new IllegalArgumentException("配置文件路径不能为空");
    }

    // 使用 try-with-resources
    try (InputStream in = getClass().getResourceAsStream(path)) {
        if (in == null) {
            throw new FileNotFoundException("配置文件不存在: " + path);
        }
        return new String(in.readAllBytes(), StandardCharsets.UTF_8);
    } catch (IOException e) {
        throw new RuntimeException("读取配置文件失败: " + path, e);
    }
}
```

## 异常处理与函数式编程

```java
// 使用 try-catch in lambda（Java 8+）
List<String> results = names.stream()
    .map(name -> {
        try {
            return parseName(name);
        } catch (ParseException e) {
            return "UNKNOWN";
        }
    })
    .collect(Collectors.toList());

// 使用 wrapException 工具
public static <T> Function<T, R> wrapException(
        ThrowingFunction<T, R> function) {
    return t -> {
        try {
            return function.apply(t);
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    };
}

@FunctionalInterface
public interface ThrowingFunction<T, R> {
    R apply(T t) throws Exception;
}

// 使用 Optional 处理可能的 null
Optional.ofNullable(user)
    .map(User::getEmail)
    .filter(email -> email.contains("@"))
    .orElse("default@example.com");
```

## 异常测试

```java
// 测试抛出预期异常
@Test(expected = IllegalArgumentException.class)
public void testInvalidInput() {
    validator.validate(null);
}

// 测试异常消息
@Test
public void testExceptionMessage() {
    IllegalArgumentException e = assertThrows(
        IllegalArgumentException.class,
        () -> validator.validate(null)
    );
    assertEquals("输入不能为空", e.getMessage());
}

// 测试异常链
@Test
public void testExceptionChain() {
    RuntimeException e = assertThrows(
        RuntimeException.class,
        () -> service.process()
    );
    assertTrue(e.getCause() instanceof IOException);
    assertEquals("原始错误", e.getCause().getMessage());
}
```
