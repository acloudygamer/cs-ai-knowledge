## 列表 vs 生成器

```python
import time

# 列表（立即计算，占内存）
start = time.time()
squares = [x**2 for x in range(10**7)]
elapsed = time.time() - start
print(f"List: {elapsed:.2f}s, Memory: {len(squares) * 8 / 1024 / 1024:.1f}MB")

# 生成器（惰性求值，省内存）
start = time.time()
gen = (x**2 for x in range(10**7))
# 遍历时才计算
total = sum(gen)
elapsed = time.time() - start
print(f"Generator: {elapsed:.2f}s")

# itertools 处理大数据
import itertools

# 读取大文件
def read_large_file(filename):
    with open(filename, 'r') as f:
        for line in f:
            yield line.strip()

# 高效的链式操作
data = range(10**7)
result = sum(x for x in data if x % 2 == 0)  # 不创建中间列表

# 内存对比
import sys

large_list = [x for x in range(1000000)]
large_gen = (x for x in range(1000000))

print(sys.getsizeof(large_list))  # ~8MB
print(sys.getsizeof(large_gen))   # ~200 bytes
```

## Python 3.11 性能提升

Python 3.11 在 Faster CPython 项目中实现了显著的性能提升，相比 Python 3.10 平均快 25%：

### 自适应解释器

Python 3.11 引入了自适应解释器，可以根据运行时信息优化字节码：

```python
# Python 3.11 的解释器会分析运行时类型
# 对热点代码进行专门优化
def compute(data):
    result = 0
    for item in data:
        result += item  # 多次执行后，解释器假设 item 是 int
    return result

# 首次执行：解释执行
# 热点代码：生成立即执行（quickened）字节码，绕过解释器
```

### 帧堆栈优化

Python 3.11 减少了帧对象的内存分配和访问开销：

```python
# Python 3.10 及之前：每个函数调用创建新帧对象
def outer():
    x = 1
    def inner():
        return x + 1
    return inner()

# Python 3.11：帧复用，减少内存分配
# 内联栈帧访问，避免重复查找
```

### 零成本异常

Python 3.11 重写了异常处理机制：

```python
# 之前：try 块有轻微性能开销
def old_pattern():
    start = time.time()
    result = None
    for i in range(1000000):
        try:
            result = data[i]
        except IndexError:
            result = default
    return result

# Python 3.11：try 块几乎零开销，只有实际抛异常时才付出代价
def new_pattern():
    start = time.time()
    result = None
    for i in range(1000000):
        try:
            result = data[i]
        except KeyError:
            result = default
    return result
```

### 快速识别热点的技巧

```python
import time

# 方法1：简单计时
start = time.perf_counter()
hot_code()
print(f"Time: {time.perf_counter() - start:.3f}s")

# 方法2：使用上下文管理器
from contextlib import contextmanager

@contextmanager
def timer(name):
    start = time.perf_counter()
    yield
    print(f"{name}: {time.perf_counter() - start:.3f}s")

with timer("Data processing"):
    process_data()

# 方法3：使用 functools.wraps 保存被装饰函数元数据
import functools
import time

def timed(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__}: {time.perf_counter() - start:.3f}s")
        return result
    return wrapper
```

## __slots__ 优化内存

`__slots__` 是 Python 3.11 优化内存的重要手段：

```python
import sys

# 普通类：每个实例有 __dict__，占用大量内存
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
print(sys.getsizeof(p))              # 约 48 bytes（不含 __dict__ 内容）
print(sys.getsizeof(p.__dict__))     # 约 104 bytes（实例属性字典）

# 使用 __slots__：无 __dict__，内存占用大幅减少
class PointSlotted:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y

p2 = PointSlotted(1, 2)
print(sys.getsizeof(p2))              # 约 40 bytes（更小）

# __slots__ 的额外优点：属性访问更快
# Python 3.11 中 __slots__ 的查找性能进一步提升
```

### __slots__ 最佳实践

```python
# 1. 基础用法
class Simple:
    __slots__ = ('x', 'y', 'z')

# 2. 可继承的写法
class Base:
    __slots__ = ()

class Derived(Base):
    __slots__ = ('name',)

# 3. 结合弱引用
import weakref

class CacheEntry:
    __slots__ = ('key', 'value', '_weak_ref')

    def __init__(self, key, value):
        self.key = key
        self.value = value
        self._weak_ref = weakref.ref(value)

# 4. 配合 dataclasses（Python 3.10+）
from dataclasses import dataclass

@dataclass(slots=True)
class DataPoint:
    x: float
    y: float
    label: str

# 5. 注意多继承时的 __slots__
class A:
    __slots__ = ('a',)

class B:
    __slots__ = ('b',)

class C(A, B):  # 正常，包含 A 和 B 的 slots
    __slots__ = ('c',)
```

## 常用性能优化模式

### 避免全局变量查找

```python
# 慢：全局变量每次都要查找
def slow_sum():
    total = 0
    for i in range(1000000):
        total += i  # 每次都要查全局 total
    return total

# 快：使用局部变量
def fast_sum():
    total = 0
    local_total = total  # 局部变量更快
    for i in range(1000000):
        local_total += i
    return local_total

# 最快：Python 3 的 sum 本身已经高度优化
result = sum(range(1000000))
```

### 局部变量访问优化

```python
# 多次访问同一属性时，缓存到局部变量
class DataProcessor:
    def process(self, items):
        # 慢：每次 self.data 都要查找
        result = []
        for item in items:
            result.append(self.data[item])
        return result

    def process_fast(self, items):
        # 快：缓存到局部变量
        data = self.data  # 一次查找
        result = []
        for item in items:
            result.append(data[item])
        return result
```

### 减少函数调用开销

```python
# 列表推导式 vs map
import timeit

# 列表推导式（通常更快）
t1 = timeit.timeit('[x**2 for x in range(1000)]', number=10000)

# map + lambda（略慢）
t2 = timeit.timeit('list(map(lambda x: x**2, range(1000)))', number=10000)

# 内置函数（最快）
t3 = timeit.timeit('[x*x for x in range(1000)]', number=10000)  # 避免幂运算
```

### 字符串拼接优化

```python
# 慢：字符串相加
result = ""
for i in range(10000):
    result += str(i)

# 快：列表 join
parts = []
for i in range(10000):
    parts.append(str(i))
result = ''.join(parts)

# 最快：列表推导式 + join
result = ''.join(str(i) for i in range(10000))
```
