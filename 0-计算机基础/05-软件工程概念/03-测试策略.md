# 测试策略

## 概念

**测试**是验证软件正确性的手段，分为多个层次，从单元到端到端。

```
┌─────────────────────────────────────────────┐
│           端到端测试 (E2E)                 │
│   模拟真实用户操作，整系统验证              │
├─────────────────────────────────────────────┤
│           集成测试 (Integration)           │
│   验证模块间协作，如API、数据库交互         │
├─────────────────────────────────────────────┤
│           单元测试 (Unit)                  │
│   测试独立函数/类，最小粒度                │
└─────────────────────────────────────────────┘
```

## 关系

**关键连接**：
- 单元测试 → **代码**：测试最小可测单元
- 集成测试 → **模块**：测试多个单元协作
- E2E测试 → **系统**：测试完整用户流程
- 测试 → **CI/CD**：自动化测试是CI/CD的核心
- 测试 → **重构**：测试保障重构安全性

## 测试金字塔

```
                    ╱╲
                   ╱  ╲
                  ╱ E2E╲        ← 少量，慢，贵
                 ╱──────╲
                ╱Integration╲   ← 中量
               ╱────────────╲
              ╱  Unit Tests  ╲  ← 大量，快，便宜
             ╱──────────────╲
```

**核心原则**：
- 底层测试多：单元测试快速、便宜、隔离
- 顶层测试少：E2E测试慢、贵、不稳定

## 单元测试

### Python pytest
```python
# test_calculator.py
import pytest
from calculator import add, divide

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

def test_divide():
    assert divide(6, 2) == 3
    with pytest.raises(ValueError):
        divide(1, 0)

# 运行
pytest test_calculator.py -v
```

### JavaScript Jest
```javascript
// calculator.test.js
const { add, divide } = require('./calculator');

test('adds two numbers', () => {
    expect(add(1, 2)).toBe(3);
});

test('throws on divide by zero', () => {
    expect(() => divide(1, 0)).toThrow();
});

// 运行
npx jest
```

## 集成测试

### API 集成测试
```python
# test_api.py
import requests
import pytest

BASE_URL = "http://localhost:3000/api"

def test_create_user():
    response = requests.post(f"{BASE_URL}/users", json={
        "name": "Alice",
        "email": "alice@example.com"
    })
    assert response.status_code == 201
    data = response.json()
    assert data["name"] == "Alice"
    assert "id" in data

def test_get_user():
    # 先创建
    create_resp = requests.post(f"{BASE_URL}/users", json={
        "name": "Bob"
    })
    user_id = create_resp.json()["id"]

    # 再获取
    get_resp = requests.get(f"{BASE_URL}/users/{user_id}")
    assert get_resp.status_code == 200
    assert get_resp.json()["name"] == "Bob"
```

### 数据库集成测试
```python
# test_with_db.py
import pytest
from sqlalchemy import create_engine
from models import User

@pytest.fixture
def db_session():
    engine = create_engine('sqlite:///:memory:')
    User.metadata.create_all(engine)
    session = Session(bind=engine)
    yield session
    session.close()

def test_create_user(db_session):
    user = User(name="Alice", email="alice@test.com")
    db_session.add(user)
    db_session.commit()

    assert db_session.query(User).count() == 1
```

## 端到端测试 (E2E)

### Playwright
```python
# test_e2e.py
from playwright.sync_api import sync_playwright

def test_login_flow():
    with sync_playwright() as p:
        browser = p.chromium.launch()
        page = browser.new_page()

        page.goto("http://localhost:3000/login")
        page.fill("#email", "user@example.com")
        page.fill("#password", "password123")
        page.click("button[type='submit']")

        page.wait_for_url("**/dashboard")
        assert page.locator("h1").text_content() == "Welcome"

        browser.close()
```

### Cypress
```javascript
// cypress/e2e/login.cy.js
describe('Login', () => {
    it('should login successfully', () => {
        cy.visit('/login');
        cy.get('#email').type('user@example.com');
        cy.get('#password').type('password123');
        cy.get('button[type="submit"]').click();
        cy.url().should('include', '/dashboard');
    });
});
```

## TDD 测试驱动开发

### TDD 流程
```
1. 写一个失败的测试 (Red)    → 运行测试，应失败
2. 写最小实现通过测试 (Green) → 只写能通过的内容
3. 重构 (Refactor)           → 改善代码，保持测试通过
```

### TDD 示例 (JavaScript)
```javascript
// Step 1: 先写测试（此时 Calculator 类不存在）
// calculator.test.js
describe('Calculator', () => {
    it('should add two numbers', () => {
        const calc = new Calculator();
        expect(calc.add(1, 2)).toBe(3);
    });

    it('should handle negative numbers', () => {
        const calc = new Calculator();
        expect(calc.add(-1, -2)).toBe(-3);
    });
});

// Step 2: 运行测试 → 失败 (Class not found)

// Step 3: 写最小实现
class Calculator {
    add(a, b) {
        return a + b;
    }
}

// Step 4: 测试通过后，重构（如需要）
// 加入边界检查、错误处理等
```

## BDD 行为驱动开发

### Gherkin 语法 (Cucumber)
```gherkin
# features/login.feature
Feature: User Login
  Scenario: Valid user can login
    Given I am on the login page
    When I enter "user@example.com" and "password123"
    And I click the "Login" button
    Then I should be redirected to the dashboard
    And I should see "Welcome, User"

  Scenario: Invalid credentials show error
    Given I am on the login page
    When I enter "invalid@example.com" and "wrongpass"
    And I click the "Login" button
    Then I should see an error message "Invalid credentials"
```

### Step Definitions (Python)
```python
# steps/login_steps.py
from behave import given, when, then

@given('I am on the login page')
def step_open_login(context):
    context.browser.get('http://localhost:3000/login')

@when('I enter "{email}" and "{password}"')
def step_enter_credentials(context, email, password):
    context.browser.fill('#email', email)
    context.browser.fill('#password', password)

@when('I click the "{button}" button')
def step_click_button(context, button):
    context.browser.click(f'button:contains("{button}")')

@then('I should see an error message "{message}"')
def step_see_error(context, message):
    assert context.browser.is_text_present(message)
```

## Mock 与 Stub

### Python unittest.mock
```python
from unittest.mock import Mock, patch

def test_fetch_user(mocker):
    # Mock外部API调用
    mock_get = mocker.patch('requests.get')
    mock_get.return_value.json.return_value = {
        'id': 1, 'name': 'Alice'
    }

    user = fetch_user(1)
    assert user.name == 'Alice'
    mock_get.assert_called_once_with('http://api/users/1')
```

### Jest Mock
```javascript
// Mock模块
jest.mock('./api', () => ({
    fetchUser: jest.fn().mockResolvedValue({ id: 1, name: 'Alice' })
}));

// Mock函数
test('fetches user', async () => {
    const user = await fetchUser(1);
    expect(user.name).toBe('Alice');
});
```

            - uses: actions/setup-node@v3

            - run: npm run test:e2e
