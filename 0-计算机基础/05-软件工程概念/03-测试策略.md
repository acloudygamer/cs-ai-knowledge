# 测试策略

## 概念

**测试**是验证软件正确性的手段，分为多个层次，从单元到端到端。

```
┌─────────────────────────────────────────────┐
│           端到端测试 (E2E)                 │
│   模拟真实用户操作，整系统验证              │
├─────────────────────────────────────────────┤
│           集成测试 (Integration)           │
│   验证模块间协作，如API、数据库交互         │
├─────────────────────────────────────────────┤
│           单元测试 (Unit)                  │
│   测试独立函数/类，最小粒度                │
└─────────────────────────────────────────────┘
```

## 关系

**关键连接**：
- 单元测试 → **代码**：测试最小可测单元
- 集成测试 → **模块**：测试多个单元协作
- E2E测试 → **系统**：测试完整用户流程
- 测试 → **CI/CD**：自动化测试是CI/CD的核心
- 测试 → **重构**：测试保障重构安全性

## 测试金字塔

```
                    ╱╲
                   ╱  ╲
                  ╱ E2E╲        ← 少量，慢，贵
                 ╱──────╲
                ╱Integration╲   ← 中量
               ╱────────────╲
              ╱  Unit Tests  ╲  ← 大量，快，便宜
             ╱──────────────╲
```

**核心原则**：
- 底层测试多：单元测试快速、便宜、隔离
- 顶层测试少：E2E测试慢、贵、不稳定

## 单元测试

### Python pytest
```python
# test_calculator.py
import pytest
from calculator import add, divide

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

def test_divide():
    assert divide(6, 2) == 3
    with pytest.raises(ValueError):
        divide(1, 0)

# 运行
pytest test_calculator.py -v
```

### JavaScript Jest
```javascript
// calculator.test.js
const { add, divide } = require('./calculator');

test('adds two numbers', () => {
    expect(add(1, 2)).toBe(3);
});

test('throws on divide by zero', () => {
    expect(() => divide(1, 0)).toThrow();
});

// 运行
npx jest
```

## 集成测试

### API 集成测试
```python
# test_api.py
import requests
import pytest

BASE_URL = "http://localhost:3000/api"

def test_create_user():
    response = requests.post(f"{BASE_URL}/users", json={
        "name": "Alice",
        "email": "alice@example.com"
    })
    assert response.status_code == 201
    data = response.json()
    assert data["name"] == "Alice"
    assert "id" in data

def test_get_user():
    # 先创建
    create_resp = requests.post(f"{BASE_URL}/users", json={
        "name": "Bob"
    })
    user_id = create_resp.json()["id"]

    # 再获取
    get_resp = requests.get(f"{BASE_URL}/users/{user_id}")
    assert get_resp.status_code == 200
    assert get_resp.json()["name"] == "Bob"
```

### 数据库集成测试
```python
# test_with_db.py
import pytest
from sqlalchemy import create_engine
from models import User

@pytest.fixture
def db_session():
    engine = create_engine('sqlite:///:memory:')
    User.metadata.create_all(engine)
    session = Session(bind=engine)
    yield session
    session.close()

def test_create_user(db_session):
    user = User(name="Alice", email="alice@test.com")
    db_session.add(user)
    db_session.commit()

    assert db_session.query(User).count() == 1
```

## 端到端测试 (E2E)

### Playwright
```python
# test_e2e.py
from playwright.sync_api import sync_playwright

def test_login_flow():
    with sync_playwright() as p:
        browser = p.chromium.launch()
        page = browser.new_page()

        page.goto("http://localhost:3000/login")
        page.fill("#email", "user@example.com")
        page.fill("#password", "password123")
        page.click("button[type='submit']")

        page.wait_for_url("**/dashboard")
        assert page.locator("h1").text_content() == "Welcome"

        browser.close()
```

### Cypress
```javascript
// cypress/e2e/login.cy.js
describe('Login', () => {
    it('should login successfully', () => {
        cy.visit('/login');
        cy.get('#email').type('user@example.com');
        cy.get('#password').type('password123');
        cy.get('button[type="submit"]').click();
        cy.url().should('include', '/dashboard');
    });
});
```

## Mock 与 Stub

### Python unittest.mock
```python
from unittest.mock import Mock, patch

def test_fetch_user(mocker):
    # Mock外部API调用
    mock_get = mocker.patch('requests.get')
    mock_get.return_value.json.return_value = {
        'id': 1, 'name': 'Alice'
    }

    user = fetch_user(1)
    assert user.name == 'Alice'
    mock_get.assert_called_once_with('http://api/users/1')
```

### Jest Mock
```javascript
// Mock模块
jest.mock('./api', () => ({
    fetchUser: jest.fn().mockResolvedValue({ id: 1, name: 'Alice' })
}));

// Mock函数
test('fetches user', async () => {
    const user = await fetchUser(1);
    expect(user.name).toBe('Alice');
});
```

## CI/CD 中的测试

```yaml
# .github/workflows/test.yml
name: Test

on: [push, pull_request]

jobs:
    test:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v3
            - uses: actions/setup-node@v3
              with:
                  node-version: '18'
            - run: npm ci
            - run: npm test
            - run: npm run test:e2e
```

## 面试要点

1. **测试金字塔**：单元/集成/E2E的比例和原因
2. **单元测试**：孤立性、快速、覆盖边界情况
3. **集成测试**：模块间协作、数据库、API测试
4. **E2E测试**：完整流程、Playwright/Cypress
5. **Mock**：为什么要Mock，什么需要Mock
6. **TDD**：先写测试的开发方式
