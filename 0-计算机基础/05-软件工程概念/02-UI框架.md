# UI框架

## 概念

**UI框架**是用于构建用户界面的工具集，涵盖桌面、移动端、网页等平台。

```
┌─────────────────────────────────────┐
│            UI 框架                    │
├─────────────────────────────────────┤
│  Web: React / Vue / Angular         │
│  Desktop: Electron / Qt / WPF       │
│  Mobile: UIKit / SwiftUI / Flutter  │
└─────────────────────────────────────┘
```

## 关系

**关键连接**：
- UI框架 → **操作系统**：框架调用系统原生UI组件
- UI框架 → **渲染引擎**：Web框架依赖浏览器渲染
- UI框架 ← **前端**：Web UI是前端开发的核心
- UI框架 ← **移动开发**：iOS/Android原生开发

## Web UI 框架

### React
```jsx
// React 组件
function Counter() {
    const [count, setCount] = useState(0);

    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
        </div>
    );
}

// 状态管理
const store = create((set) => ({
    count: 0,
    increment: () => set((state) => ({ count: state.count + 1 }))
}));
```

### Vue
```vue
<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">Increment</button>
  </div>
</template>

<script setup>
import { ref } from 'vue';
const count = ref(0);
const increment = () => count.value++;
</script>
```

### 对比

| 框架 | 特点 | 适用场景 |
|------|------|----------|
| React | 组件化、虚拟DOM、生态大 | 中大型应用 |
| Vue | 上手简单、双向绑定 | 快速开发 |
| Angular | 完整解决方案、TypeScript优先 | 企业级 |
| Svelte | 编译时优化、无虚拟DOM | 轻量应用 |

## 桌面 UI

### Electron
```javascript
// Electron 主进程
const { app, BrowserWindow } = require('electron');

app.whenReady().then(() => {
    const win = new BrowserWindow({
        width: 1200,
        height: 800,
        webPreferences: { nodeIntegration: true }
    });
    win.loadFile('index.html');
});
```

**Electron = Chromium + Node.js + 系统原生API**

| 优点 | 缺点 |
|------|------|
| Web技术跨平台 | 打包体积大 (100MB+) |
| 生态成熟 | 内存占用高 |
| 开发快速 | 不如原生性能 |

### Qt (C++)
```cpp
// Qt 按钮示例
#include <QApplication>

int main(int argc, char *argv[]) {
    QApplication app(argc, argv);

    QPushButton button("Hello, Qt!");
    button.resize(200, 100);
    button.show();

    return app.exec();
}
```

## 组件化设计

现代UI框架都支持组件化：

```jsx
// 可复用组件
function Button({ children, variant = 'primary', onClick }) {
    return (
        <button
            className={`btn btn-${variant}`}
            onClick={onClick}
        >
            {children}
        </button>
    );
}

// 使用
<Button variant="secondary" onClick={handleCancel}>
    Cancel
</Button>
```

### 组件设计原则
```jsx
// 1. 单一职责：每个组件只做一件事
function UserAvatar({ userId, size }) {
    const [user] = useUser(userId);  // 数据获取单独处理
    return <img src={user.avatar} className={`avatar-${size}`} />;
}

// 2. 受控组件 vs 非受控组件
// 受控：状态由 React 控制
function ControlledInput({ value, onChange }) {
    return <input value={value} onChange={onChange} />;
}

// 非受控：状态由 DOM 自身控制
function UncontrolledInput({ defaultValue }) {
    const inputRef = useRef();
    return <input defaultValue={defaultValue} ref={inputRef} />;
}

// 3. 复合组件模式
function Tabs({ children }) {
    const [activeTab, setActiveTab] = useState(0);
    return (
        <TabContext.Provider value={{ activeTab, setActiveTab }}>
            {children}
        </TabContext.Provider>
    );
}

Tabs.Tab = function Tab({ label, children }) {
    const { activeTab, setActiveTab } = useContext(TabContext);
    return <div onClick={() => setActiveTab(label)}>{children}</div>;
};
```

### React 状态管理
```jsx
// Context API - 跨层级状态共享
const ThemeContext = createContext('light');

function App() {
    const [theme, setTheme] = useState('dark');
    return (
        <ThemeContext.Provider value={{ theme, setTheme }}>
            <Dashboard />
        </ThemeContext.Provider>
    );
}

// useContext 消费
function Sidebar() {
    const { theme } = useContext(ThemeContext);
    return <div className={`sidebar-${theme}`}>...</div>;
}

// 第三方状态管理 (Redux/Zustand)
import { create } from 'zustand';

const useStore = create((set) => ({
    count: 0,
    increment: () => set((state) => ({ count: state.count + 1 })),
    decrement: () => set((state) => ({ count: state.count - 1 })),
}));

// 使用
function Counter() {
    const { count, increment } = useStore();
    return <button onClick={increment}>{count}</button>;
}
```

## 样式方案对比

| 方案 | 特点 | 代表框架/库 |
|------|------|------------|
| CSS Modules | 局部作用域，无运行时开销 | 原生支持 |
| Styled Components | 模板字符串，动态样式 | styled-components |
| Tailwind CSS | 工具类原子CSS | Tailwind |
| CSS-in-JS | JS写样式，主题系统强 | Emotion, JSS |

```html
    Click me

<!-- 响应式设计 -->
<div class="grid grid-cols-1 md:grid-cols-3 lg:grid-cols-4 gap-4">
    <!-- 移动端1列，平板3列，桌面4列 -->
</div>
```

## 面试要点

1. **Web框架对比**：React vs Vue vs Angular特点
2. **跨平台方案**：WebView vs 原生渲染 vs 自绘
3. **Electron原理**：Chromium + Node.js，无原生性能
4. **移动框架**：UIKit vs SwiftUI vs Jetpack Compose
5. **Flutter**：自绘引擎Skia，Dart语言
6. **组件化**：组件设计原则，状态管理
7. **虚拟DOM**：React虚拟DOM diff算法原理
8. **响应式原理**：Vue响应式 vs React状态驱动
9. **样式方案**：CSS Modules、Tailwind、styled-components 优劣
