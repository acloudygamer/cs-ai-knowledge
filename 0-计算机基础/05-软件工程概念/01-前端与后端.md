# 前端与后端

## 概念

**前端**是用户直接交互的部分，**后端**是服务器端处理数据和业务逻辑的部分。

```
┌─────────────────────────────────────────────────┐
│                   用户                          │
└────────────────────┬──────────────────────────┘
                     │ 浏览器/App
┌────────────────────▼──────────────────────────┐
│              前端 (Frontend)                    │
│   HTML/CSS/JS 或 React/Vue/Flutter            │
└────────────────────┬──────────────────────────┘
                     │ HTTP/REST/gRPC
┌────────────────────▼──────────────────────────┐
│              后端 (Backend)                    │
│   Python/Go/Java/Node.js + 数据库            │
└────────────────────┬──────────────────────────┘
                     │
              ┌──────▼──────┐
              │   数据库     │
              │ MySQL/Redis  │
              └─────────────┘
```

## 关系

**关键连接**：
- 前端 → **后端**：通过API请求获取数据
- 后端 → **数据库**：查询和存储数据
- 后端 → **缓存**：使用Redis等提高性能
- 前端 → **CDN**：加载静态资源
- 全栈 → **前端+后端**：一个开发者掌握两端

### REST API 设计
```javascript
// Express.js REST API
const express = require('express');
const app = express();

app.get('/api/users/:id', async (req, res) => {
    const user = await db.users.findById(req.params.id);
    if (!user) {
        return res.status(404).json({ error: 'User not found' });
    }
    res.json(user);
});

app.post('/api/users', async (req, res) => {
    const user = await db.users.create(req.body);
    res.status(201).json(user);
});

app.listen(3000);
```

## 认证机制

### JWT (JSON Web Token)
```javascript
// 后端签发 JWT
const jwt = require('jsonwebtoken');
const token = jwt.sign(
    { userId: user.id, role: 'admin' },
    process.env.JWT_SECRET,
    { expiresIn: '7d' }
);

// 前端存储和使用
// localStorage.setItem('token', token);
// axios.defaults.headers.common['Authorization'] = `Bearer ${token}`;

// 后端验证 JWT
const authenticate = (req, res, next) => {
    const token = req.headers.authorization?.split(' ')[1];
    if (!token) return res.status(401).json({ error: 'Unauthorized' });

    try {
        const decoded = jwt.verify(token, process.env.JWT_SECRET);
        req.user = decoded;
        next();
    } catch (err) {
        res.status(403).json({ error: 'Invalid token' });
    }
};

// JWT 结构: header.payload.signature
// eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjF9.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```

### Session vs JWT 对比
| 特性 | Session | JWT |
|------|---------|-----|
| 存储位置 | 服务端内存/Redis | 客户端 (无状态) |
| 扩展性 | 需共享session | 无状态，易扩展 |
| 安全性 | 依赖cookie HttpOnly | 需防止XSS窃取 |
| 适用场景 | 有状态服务 | 微服务、跨域API |
| 撤销能力 | 可立即撤销 | 需黑名单机制 |

### OAuth2 授权流程
```
用户 → 第三方登录 → 授权服务器 → 返回授权码 → 换取 Access Token
                                   ↓
                          资源服务器验证Token
                                   ↓
                          返回用户数据
```

## 前后端分离

### 传统 vs 分离
| 模式 | 前端 | 后端 | 通信 |
|------|------|------|------|
| 传统 | JSP/PHP模板 | 同一进程 | 页面渲染 |
| 前后端分离 | SPA + API | REST API | JSON数据 |

### 分离优势
1. **独立开发**：前后端可并行开发
2. **独立部署**：前端静态资源可放CDN
3. **技术灵活**：前端用React，后端用Go
4. **移动支持**：同一API供App使用

## 全栈开发

// Node.js + Express 全栈示例

const express = require('express');

## 部署与运维

### 前端部署 (CDN + CDN边缘加速)
```bash
# 构建生产版本
npm run build
# 输出到 dist/ 目录，可部署到 CDN

# Nginx 配置
server {
    listen 80;
    server_name myapp.com;
    root /var/www/dist;
    index index.html;

    # 静态资源缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API 代理到后端
    location /api/ {
        proxy_pass http://backend:3000/;
    }
}
```

### 后端部署
```bash
# PM2 进程管理 (Node.js)
pm2 start server.js --name api -i 4  # 4个实例负载均衡
pm2 list
pm2 logs api
pm2 restart api

# Docker 部署
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
```

## 性能优化

### 前端优化
- **代码分割**：React.lazy + Suspense 懒加载路由
- **CDN加速**：静态资源放到CDN
- **图片优化**：WebP格式、响应式图片 srcset
- **懒加载**：Intersection Observer 延迟加载
- **缓存**：Service Worker 离线缓存

### 后端优化
- **缓存层**：Redis 缓存热点数据
- **数据库**：索引优化、查询优化
- **连接池**：数据库连接复用
- **异步处理**：消息队列解耦耗时任务

```javascript
// Redis 缓存示例
const redis = require('redis');
const client = redis.createClient();

async function getUserWithCache(userId) {
    const cacheKey = `user:${userId}`;

    // 先查缓存
    const cached = await client.get(cacheKey);
    if (cached) {
        return JSON.parse(cached);
    }

    // 缓存未命中，查数据库
    const user = await db.users.findById(userId);

    // 写入缓存，过期时间 5 分钟
    await client.setEx(cacheKey, 300, JSON.stringify(user));

    return user;
}
```
