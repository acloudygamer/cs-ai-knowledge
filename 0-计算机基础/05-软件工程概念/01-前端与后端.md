# 前端与后端

## 概念

**前端**是用户直接交互的部分，**后端**是服务器端处理数据和业务逻辑的部分。

```
┌─────────────────────────────────────────────────┐
│                   用户                          │
└────────────────────┬──────────────────────────┘
                     │ 浏览器/App
┌────────────────────▼──────────────────────────┐
│              前端 (Frontend)                    │
│   HTML/CSS/JS 或 React/Vue/Flutter            │
└────────────────────┬──────────────────────────┘
                     │ HTTP/REST/gRPC
┌────────────────────▼──────────────────────────┐
│              后端 (Backend)                    │
│   Python/Go/Java/Node.js + 数据库            │
└────────────────────┬──────────────────────────┘
                     │
              ┌──────▼──────┐
              │   数据库     │
              │ MySQL/Redis  │
              └─────────────┘
```

## 关系

**关键连接**：
- 前端 → **后端**：通过API请求获取数据
- 后端 → **数据库**：查询和存储数据
- 后端 → **缓存**：使用Redis等提高性能
- 前端 → **CDN**：加载静态资源
- 全栈 → **前端+后端**：一个开发者掌握两端

## 前端技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 结构 | HTML | 页面结构 |
| 样式 | CSS | 布局、颜色、动画 |
| 逻辑 | JavaScript/TypeScript | 交互逻辑 |
| 框架 | React/Vue/Angular | 组件化开发 |
| 移动 | React Native/Flutter | 跨平台移动App |

### React 示例
```jsx
// React 函数组件
function UserCard({ name, email }) {
    const [clicked, setClicked] = useState(false);

    return (
        <div className="card">
            <h2>{name}</h2>
            <p>{email}</p>
            <button onClick={() => setClicked(true)}>
                {clicked ? 'Clicked!' : 'Click me'}
            </button>
        </div>
    );
}
```

## 后端技术栈

| 场景 | 技术 | 特点 |
|------|------|------|
| API框架 | Express/Koa (JS), Django/Flask (Python), Gin (Go) | 轻量/全功能 |
| 企业级 | Spring Boot (Java), ASP.NET (C#) | 完整生态 |
| 序列化 | JSON/REST, Protobuf, GraphQL | 数据格式 |
| 认证 | JWT, OAuth2, Session | 安全机制 |

### REST API 设计
```javascript
// Express.js REST API
const express = require('express');
const app = express();

app.get('/api/users/:id', async (req, res) => {
    const user = await db.users.findById(req.params.id);
    if (!user) {
        return res.status(404).json({ error: 'User not found' });
    }
    res.json(user);
});

app.post('/api/users', async (req, res) => {
    const user = await db.users.create(req.body);
    res.status(201).json(user);
});

app.listen(3000);
```

## 前后端通信

### REST API
```bash
# GET 请求
curl http://api.example.com/users

# POST 请求
curl -X POST http://api.example.com/users \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice","email":"alice@example.com"}'

# 带认证
curl -H "Authorization: Bearer <token>" \
  http://api.example.com/protected
```

### GraphQL
```graphql
# 一次请求获取多个资源
query {
  user(id: 1) {
    name
    email
    posts {
      title
    }
  }
}
```

## 前后端分离

### 传统 vs 分离
| 模式 | 前端 | 后端 | 通信 |
|------|------|------|------|
| 传统 | JSP/PHP模板 | 同一进程 | 页面渲染 |
| 前后端分离 | SPA + API | REST API | JSON数据 |

### 分离优势
1. **独立开发**：前后端可并行开发
2. **独立部署**：前端静态资源可放CDN
3. **技术灵活**：前端用React，后端用Go
4. **移动支持**：同一API供App使用

## 全栈开发

```javascript
// Node.js + Express 全栈示例
// 同时处理前端和后端

const express = require('express');
const app = express();

// API路由
app.get('/api/users', (req, res) => {
    res.json([{ id: 1, name: 'Alice' }]);
});

// 前端静态文件
app.use(express.static('public'));

app.listen(3000);
```

## 面试要点

1. **前端后端区别**：职责划分、技术栈差异
2. **REST API设计**：HTTP方法(GET/POST/PUT/DELETE)、状态码
3. **前后端分离**：架构优势、API设计
4. **认证机制**：JWT vs Session vs OAuth2
5. **性能优化**：CDN、缓存、压缩、懒加载
6. **全栈能力**：前端、后端、数据库都能处理
