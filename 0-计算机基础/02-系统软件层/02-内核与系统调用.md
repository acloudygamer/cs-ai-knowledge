# 内核与系统调用

## 概念

**内核**是操作系统的核心，运行在特权模式，直接管理硬件。**系统调用**是用户程序请求内核服务的接口。

```
用户程序
    ↓ (系统调用)
内核空间 ←→ 硬件
```

## 关系

**关键连接**：
- 系统调用 → **内核**：用户程序通过系统调用进入内核
- 内核 → **硬件**：内核直接控制硬件资源
- 库函数 → **系统调用**：glibc等库封装系统调用为易用接口
- 容器 → **内核**：容器依赖Linux内核的namespace/cgroup隔离

## 内核职责

| 职责 | 说明 | 相关机制 |
|------|------|----------|
| 进程调度 | 分配CPU时间片 | CFS调度器 |
| 内存管理 | 虚拟内存、页面置换 | slab分配器 |
| 文件系统 | 文件的读写和管理 | VFS、ext4 |
| 设备驱动 | 与硬件通信 | 字符/块设备驱动 |
| 网络堆栈 | TCP/IP协议实现 | netfilter |

## Linux 内核结构

```
┌─────────────────────────────────────┐
│           用户空间                   │
│  glibc / 应用进程                   │
└──────────────┬──────────────────────┘
               │ 系统调用接口 (syscall)
┌──────────────▼──────────────────────┐
│              内核空间                 │
│  ┌──────────────────────────────┐   │
│  │  系统调用层 (sys_call_table)  │   │
│  └──────────────┬───────────────┘   │
│  ┌──────────────▼───────────────┐   │
│  │     内核核心 (进程调度等)     │   │
│  └──────────────┬───────────────┘   │
│  ┌──────────────▼───────────────┐   │
│  │   驱动程序 (字符/块/网络)    │   │
│  └──────────────┬───────────────┘   │
│  ┌──────────────▼───────────────┐   │
│  │     硬件 (CPU/内存/磁盘)     │   │
│  └──────────────────────────────────┘│
└─────────────────────────────────────┘
```

## 系统调用流程

```c
// 应用程序调用系统调用示意
#include <unistd.h>

// glibc封装的write函数
ssize_t write(int fd, const void* buf, size_t count) {
    // 内部触发软中断或syscall指令
    return syscall(SYS_write, fd, buf, count);
}
```

**触发系统调用的方式**：
- x86: `int 0x80` (旧) 或 `syscall` 指令 (新)
- ARM: `svc 0` 指令

## POSIX 标准

POSIX定义了Unix系统的API标准，保证程序可移植性：

```c
// POSIX标准定义的系统调用
open(), close(), read(), write()

mmap(), munmap()
socket(), connect(), accept()
```

```bash
# Linux 系统调用跟踪
strace -e trace=write ls /tmp

# 输出示例
write(1, "file1\nfile2\n", 13) = 13
```

## 常用系统调用

### 进程相关
```c
#include <unistd.h>
#include <sys/wait.h>

pid_t pid = fork();          // 创建子进程
if (pid == 0) {
    execlp("ls", "ls", NULL);  // 执行新程序
} else {
    wait(NULL);             // 等待子进程
}
```

### 文件相关
```c
#include <fcntl.h>
#include <unistd.h>

int fd = open("file.txt", O_RDONLY);  // 打开文件
char buf[100];
read(fd, buf, 100);                    // 读取
close(fd);                             // 关闭
```

### 内存相关
```c
#include <sys/mman.h>

void* addr = mmap(NULL, 4096,
    PROT_READ | PROT_WRITE,
    MAP_PRIVATE | MAP_ANONYMOUS,
    -1, 0);           // 映射内存

munmap(addr, 4096);   // 解除映射
```

## 内核态 vs 用户态

| 特性 | 内核态 | 用户态 |
|------|--------|--------|
| 特权级别 | Ring 0 (最高) | Ring 3 |
| 访问硬件 | 直接 | 受限 |
| 内存空间 | 全部 | 仅自身进程 |
| 代码崩溃 | 系统崩溃 | 可恢复 |

## 容器与内核

容器（如Docker）依赖Linux内核的隔离机制：

```bash
# Linux命名空间隔离
unshare --mount --pid --fork bash

# cgroup资源限制
cgcreate -g cpu:/limited
cgclassify -g cpu:/limited $$  # 把当前进程加入cgroup

# 查看容器的namespace
ls -l /proc/self/ns/
# 各个命名空间: pid/net/mnt/uts/ipc/user
```

**容器不是虚拟机**：
- 虚拟机：完整OS内核，完全隔离
- 容器：共享宿主机内核，轻量但隔离弱

**Linux内核命名空间**：

| 命名空间 | 隔离内容 | 相关系统调用 |
|----------|----------|--------------|
| PID | 进程ID空间 | clone(CLONE_NEWPID) |
| Network | 网络设备/端口 | clone(CLONE_NEWNET) |
| Mount | 文件系统挂载点 | clone(CLONE_NEWNS) |
| UTS | 主机名/域名 | clone(CLONE_NEWUTS) |
| IPC | 共享内存/信号量 | clone(CLONE_NEWIPC) |
| User | 用户/组ID映射 | clone(CLONE_NEWUSER) |

## VFS (虚拟文件系统)

Linux通过VFS统一支持多种文件系统：

```bash
# 查看当前挂载的文件系统
mount
df -h                     # 磁盘空间

# 查看文件系统类型
df -T
# ext4/xfs: Linux本地文件系统
# nfs/cifs: 网络文件系统
# vfat/ntfs: 兼容Windows的文件系统
```

**VFS 抽象层**：

```
应用 → 系统调用 → VFS层 → 具体文件系统(ext4/xfs/nfs)
                        ↓
              inode(索引节点) / dentry(目录项) / super_block(超级块)
```

- **inode**：文件的元数据（权限、大小、时间戳、块位置）
- **dentry**：目录项，关联文件名和inode
- **super_block**：整个文件系统的元信息

## 软中断与异常

CPU执行过程中会触发异常和中断：

```c
// 常见异常类型
除零错误 (Division by Zero)     → SIGFPE信号
非法内存访问 (Segfault)          → SIGSEGV信号
非法指令                         → SIGILL信号
```

| 类型 | 来源 | 处理方式 |
|------|------|----------|
| 硬件中断 (IRQ) | 外部设备(键盘/网卡) | 中断处理程序，立即响应 |
| 软中断 | 软件触发(system call) | 慢速路径，内核处理 |
| 异常 | CPU执行出错 | 异常处理，可能杀死进程 |

## 中断处理流程

```
硬件中断 → IRQ向量表 → 中断描述符(IDT)
                         ↓
              保存上下文 → 调用中断处理程序 → 恢复上下文
```

```bash
# 查看中断使用情况
cat /proc/interrupts
# CPU0: IRQ号, 硬件中断, 设备名称
# LOC: 本地时钟中断
# NMI: 非屏蔽中断
```

## 内存管理进阶

Linux内存管理采用四级页表结构：

```
页全局目录(PGD) → 页上级目录(PUD) → 页中间目录(PMD) → 页表(PTE) → 物理页帧
```

```bash
# 查看进程内存映射
cat /proc/self/maps
# 地址范围 → 权限(r/w/x) → 偏移 → 设备 → inode → 文件名

# 查看内存状态
free -h                   # 物理内存使用
vmstat 1                  # 虚拟内存统计
```

| 内存区域 | 说明 |
|----------|------|
| 内核空间 | 内核代码/数据结构，永远存在 |
| 用户空间 | 进程私有，受保护 |
| 栈 | 函数调用，自动增长，向下扩展 |
| 堆 | 动态分配，手动管理，向上扩展 |
| BSS | 未初始化全局变量 |
| 数据段 | 已初始化全局变量 |
| 代码段 | 只读程序代码 |
