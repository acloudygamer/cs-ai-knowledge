# 移动OS架构

## 概念

移动操作系统（Android/iOS）是针对移动设备优化的操作系统，在内核基础上构建了完整的应用生态。

```
┌─────────────────────────────────────────────┐
│           Android / iOS 应用               │
├─────────────────────────────────────────────┤
│        Android: ART + Framework            │
│        iOS: Cocoa Touch + Objective-C/Swift │
├─────────────────────────────────────────────┤
│        Android: HAL (硬件抽象层)           │
│        iOS: Core Services                  │
├─────────────────────────────────────────────┤
│        Android: Linux Kernel              │
│        iOS: XNU Kernel (Mach + BSD)       │
├─────────────────────────────────────────────┤
│              移动设备硬件                    │
└─────────────────────────────────────────────┘
```

## 关系

**关键连接**：
- 移动OS → **桌面OS内核**：Android基于Linux，iOS基于XNU
- 移动OS → **硬件**：针对移动设备优化的功耗管理
- 移动OS → **应用框架**：各自不同的应用开发框架
- 移动OS ↔ **应用商店**：生态系统的核心分发渠道

## Android 架构

```
┌──────────────────────────────────────┐
│        应用层 (Apps)                 │
│   Java/Kotlin 代码                   │
├──────────────────────────────────────┤
│     应用框架层 (Framework)           │
│   Activity Manager / Content Provider│
│   Window Manager / Notification Mgr  │
├──────────────────────────────────────┤
│     Android 运行时 (ART)             │
│   Dalvik字节码 → 机器码             │
│   垃圾回收、线程管理                  │
├──────────────────────────────────────┤
│   硬件抽象层 (HAL)                   │
│   Camera HAL / Audio HAL / Sensor HAL │
├──────────────────────────────────────┤
│      Linux 内核 (5.x - 6.x)          │
│   Binder IPC / Ashmem / Wakelocks    │
└──────────────────────────────────────┘
```

### Android 特点

| 特性 | 说明 |
|------|------|
| 开源 | AOSP开源，厂商可定制 |
| Linux内核 | 继承Linux内核特性 |
| HAL | 抽象硬件接口，模块化 |
| ART虚拟机 | Android 5+使用AOT/JIT编译 |
| Binder | Android特有的IPC机制 |

| 沙盒 | 严格的应用隔离 |

## Android vs iOS 对比

| 维度 | Android | iOS |
|------|---------|-----|
| 内核 | Linux | XNU |
| 开发语言 | Kotlin/Java | Swift/Objective-C |
| 应用商店 | Google Play + 厂商商店 | App Store (统一) |
| 开放性 | 开源，定制灵活 | 闭源，限制多 |
| 碎片化 | 设备/系统版本多样 | 设备少，版本统一 |
| 权限管理 | 运行时请求 | 运行时请求(更严格) |
| 推送 | FCM/厂商通道 | APNs统一 |

## 面试要点

1. **Android架构**：Linux内核→HAL→ART→Framework→Apps
2. **iOS架构**：XNU→Core OS→Core Services→Media→Cocoa Touch
3. **Android vs iOS**：内核、语言、生态、权限管理差异
4. **ART vs JVM**：AOT编译、垃圾回收区别
5. **移动端优化**：后台任务、电量优化、网络优化
6. **跨平台框架**：WebView vs 原生渲染 vs 混合方案
