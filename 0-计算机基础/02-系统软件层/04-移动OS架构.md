# 移动OS架构

## 概念

移动操作系统（Android/iOS）是针对移动设备优化的操作系统，在内核基础上构建了完整的应用生态。

```
┌─────────────────────────────────────────────┐
│           Android / iOS 应用               │
├─────────────────────────────────────────────┤
│        Android: ART + Framework            │
│        iOS: Cocoa Touch + Objective-C/Swift │
├─────────────────────────────────────────────┤
│        Android: HAL (硬件抽象层)           │
│        iOS: Core Services                  │
├─────────────────────────────────────────────┤
│        Android: Linux Kernel              │
│        iOS: XNU Kernel (Mach + BSD)       │
├─────────────────────────────────────────────┤
│              移动设备硬件                    │
└─────────────────────────────────────────────┘
```

## 关系

**关键连接**：
- 移动OS → **桌面OS内核**：Android基于Linux，iOS基于XNU
- 移动OS → **硬件**：针对移动设备优化的功耗管理
- 移动OS → **应用框架**：各自不同的应用开发框架
- 移动OS ↔ **应用商店**：生态系统的核心分发渠道

## Binder IPC 机制

Binder是Android特有的进程间通信机制，效率高于传统Linux IPC：

```
进程A(客户端)                    进程B(服务端)
    │                                │
    │  Binder Driver                 │
    │◄───────────ServiceManager◄────│
    │                                │
    │──getService()──────────────────▶│ 查询服务
    │◄───────────────────────────────│返回代理
    │                                │
    │──跨进程调用─────────────────────▶│
    │◄───────────────────────────────│返回结果
```

**Binder vs Linux传统IPC**：

| 对比 | Binder | 管道/消息队列 | Socket |
|------|--------|---------------|--------|
| 性能 | 高（内存映射） | 中 | 低 |
| 双向通信 | 支持 | 受限 | 支持 |
| 安全验证 | 基于UID/PID | 无 | 无 |
| 同步/异步 | 都支持 | 通常同步 | 都支持 |

```kotlin
// AIDL (Android Interface Definition Language) 示例
// IMyService.aidl
interface IMyService {
    int getProcessId();
    void sendData(String data);
}

// Service绑定
val intent = Intent(this, MyService::class.java)
```

## Android Intent 机制

Intent是Android组件间通信的核心，用于启动Activity/Service/发送Broadcast：

```kotlin
// 显式Intent：明确指定目标组件
val intent = Intent(this, TargetActivity::class.java)
intent.putExtra("user_id", 123)
startActivity(intent)

// 隐式Intent：声明式启动，匹配IntentFilter
val intent = Intent(Intent.ACTION_VIEW).apply {
    data = Uri.parse("https://example.com")
}
startActivity(intent)

// 启动Service
val serviceIntent = Intent(this, MyService::class.java)
startService(serviceIntent)

// 发送广播
val broadcast = Intent("com.app.CUSTOM_ACTION")
sendBroadcast(broadcast)
```

**Intent过滤器**：

```xml
<!-- AndroidManifest.xml -->
<activity android:name=".MainActivity">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>

    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <data android:scheme="https" />
    </intent-filter>
</activity>
```

## ART 与 Dalvik

| 特性 | Dalvik (Android 4.4-) | ART (Android 5.0+) |
|------|----------------------|-------------------|
| 编译方式 | JIT (边运行边编译) | AOT (安装时编译) + JIT |
| 启动速度 | 快(编译在运行时) | 慢(安装时完整编译) |
| 安装后占用 | 小 | 大(预编译字节码) |
| 省电 | 一般 | 更好 |
| 垃圾回收 | 多次暂停 | 改进的并发GC |

**dex vs class**：
- JVM: .class文件，每个类一个文件
- Dalvik/ART: .dex文件，将所有类合并为一个DEX，减少IO

```bash
# 查看APK中的dex文件
unzip -l app.apk | grep dex

# APK分析工具
# apktool: 反编译资源
# jadx: 反编译dex为Java源码
# axmlprinter: 反编译AndroidManifest.xml
```

## iOS 沙盒与权限

iOS沙盒机制限制应用访问范围：

```
┌──────────────────────────────────────┐
│         iOS 应用 (沙盒内)             │
│  /var/mobile/Containers/...          │
│  只能访问:                           │
│    - 应用自身Bundle                   │
│    - 应用专属Documents/Library        │
│    - Keychain(加密存储)               │
│    - 系统分配的临时端口               │
├──────────────────────────────────────┤
│         系统框架 (授权访问)            │
│  - Camera/Microphone (需授权)         │
│  - Contacts/Photos (需授权)           │
│  - Location (需授权)                  │
├──────────────────────────────────────┤
│         其他应用 (完全隔离)            │
└──────────────────────────────────────┘
```

```swift
// iOS 权限请求
import AVFoundation
import Photos

// 相机权限
 AVCaptureDevice.requestAccess(for: .video) { granted in
    if granted {
        // 使用相机
    }
}

// 照片库访问
PHPhotoLibrary.requestAuthorization { status in
    // .authorized / .denied / .restricted
}
```

**App Store审核**：
- 每款应用必须通过苹果审核才能上架
- 审核时间通常1-3天
- 私有API使用会被拒绝
- 隐私描述(PrivacyInfo.xcprivacy)必须完整
