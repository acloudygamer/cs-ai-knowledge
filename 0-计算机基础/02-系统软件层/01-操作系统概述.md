# 操作系统概述

## 概念

**操作系统**是管理硬件资源、提供公共服务、运行应用程序的基础软件。

```
┌─────────────────────────────────────┐
│           用户应用程序               │
├─────────────────────────────────────┤
│            操作系统                  │
│  ┌───────────────────────────────┐  │
│  │  用户界面(Shell/GUI)          │  │
│  │  系统调用接口(Syscall)        │  │
│  │  内核(进程/内存/文件/设备)    │  │
│  └───────────────────────────────┘  │
├─────────────────────────────────────┤
│            硬件 (CPU/内存/磁盘)     │
└─────────────────────────────────────┘
```

## 关系

**关键连接**：
- 操作系统 → **CPU**：进程调度，决定哪个程序使用CPU
- 操作系统 → **内存**：虚拟内存管理，进程隔离
- 操作系统 → **存储**：文件系统管理
- 操作系统 → **I/O设备**：驱动程序管理设备
- 应用程序 → **操作系统**：通过系统调用请求服务

## 操作系统职能

| 职能 | 说明 | 相关概念 |
|------|------|----------|
| 进程管理 | 创建、调度、终止进程 | 进程、线程、调度器 |
| 内存管理 | 分配/回收内存，虚拟内存 | 页面置换、内存分配器 |
| 文件系统 | 管理文件的存储和访问 | inode、目录、权限 |
| 设备管理 | 驱动程序，与硬件通信 | 中断、I/O、DMA |
| 安全与权限 | 用户认证、访问控制 | 用户、组、ACL |

## 内核类型

| 类型 | 特点 | 示例 |
|------|------|------|
| **宏内核 (Monolithic)** | 所有功能在内核中，通信快 | Linux, Unix |
| **微内核 (Microkernel)** | 只保留核心功能，其他在用户空间 | Minix, QNX |
| **混合内核** | 宏内核+微内核折中 | Windows NT, macOS XNU |
| **外内核 (Exokernel)** | 极简，库操作系统 | MIT ExOS |

## 常见操作系统

| OS | 内核 | 特点 | 应用场景 |
|----|------|------|----------|
| Linux | Linux | 开源、多平台 | 服务器、嵌入式、Android |
| Windows | Windows NT | 闭源、生态好 | PC、企业 |
| macOS | XNU (Mach+BSD) | Unix兼容 | Apple设备 |
| Android | Linux | 移动端霸主 | 手机、平板 |
| iOS | XNU | 闭源、严格权限 | iPhone/iPad |

## Shell 与用户界面

Shell是用户与操作系统交互的命令行接口：

```bash
# Bash 基础命令
ls -la                  # 列出文件详情
cd /path && mkdir dir   # 切换目录并创建文件夹
cat file | grep "pattern" | sort  # 管道组合命令

# 环境变量
echo $PATH              # 查看PATH变量
export VAR=value        # 设置环境变量
env                     # 显示所有环境变量

# 输入输出重定向
command > output.txt    # 标准输出重定向到文件
command 2>&1           # 错误输出重定向到标准输出
command < input.txt     # 从文件读取输入
```

```bash
# 进程前后台控制
./my_program &          # 后台运行程序
jobs                    # 查看后台任务
fg %1                   # 将后台任务切回前台
Ctrl+Z                  # 挂起当前任务
bg %1                   # 后台继续运行挂起任务
```

## OS 启动流程

计算机启动经历多个阶段：

```
BIOS/UEFI → 引导扇区 → 内核加载 → init/systemd → 用户空间
    ↓           ↓           ↓           ↓              ↓
  硬件自检    MBR/GPT    内存初始化   启动系统服务    登录界面
```

```bash
# Linux 查看启动日志
dmesg                    # 内核环缓冲区消息
journalctl -b            # systemd本次启动日志
systemctl list-units     # 查看所有系统服务单元
```

**BIOS vs UEFI**：
- BIOS：传统固件，使用MBR分区表，最大2TB磁盘限制
- UEFI：现代固件，使用GPT分区表，支持安全启动(Secure Boot)

## 虚拟化与容器

操作系统还支持虚拟化和隔离技术：

```bash
# Docker 容器（共享宿主机内核）
docker run -it ubuntu bash    # 启动Ubuntu容器
docker ps -a                  # 查看所有容器
docker stop container_id       # 停止容器

# 虚拟机 vs 容器
# 虚拟机：QEMU/KVM/VirtualBox，每个VM有独立内核，完全隔离
# 容器：Docker/Podman，共享宿主机内核，轻量级隔离
```

| 特性 | 虚拟机 | 容器 |
|------|--------|------|
| 隔离级别 | 硬件级 | OS级 |
| 启动速度 | 慢 (分钟级) | 快 (秒级) |
| 资源开销 | 大 (完整OS) | 小 (共享内核) |
| 迁移 | 整VM镜像 | 镜像层共享 |

## 系统调用 (System Call)

应用程序通过系统调用请求OS服务：

```c
// Linux 系统调用示例
#include <unistd.h>

int main() {
    // write 是系统调用
    const char* msg = "Hello, OS!\n";
    write(STDOUT_FILENO, msg, 13);  // sys_write

    // read 是系统调用
    char buf[100];
    read(STDIN_FILENO, buf, 100);   // sys_read

    // exit 也是系统调用
    _exit(0);                        // sys_exit
}
```

**系统调用 vs 函数调用**：

| 调用类型 | 执行位置 | 特权级别 |
|----------|----------|----------|
| 普通函数调用 | 用户空间 | 用户态 |
| 系统调用 | 内核空间 | 内核态 |

## 进程与程序

```python
# 进程是程序的运行实例
process = {
    "程序": "静态文件 (磁盘上的可执行文件)",
    "进程": "运行中的实例 (内存中的指令和数据)",
    "线程": "进程内的执行单元，共享进程资源"
}
```

```bash
# Linux 查看进程
ps aux           # 列出所有进程
top              # 实时查看系统状态
htop             # 更友好的交互式进程查看

# Windows
tasklist         # 列出进程
taskmgr          # 任务管理器
```
