# 操作系统概述

## 概念

**操作系统**是管理硬件资源、提供公共服务、运行应用程序的基础软件。

```
┌─────────────────────────────────────┐
│           用户应用程序               │
├─────────────────────────────────────┤
│            操作系统                  │
│  ┌───────────────────────────────┐  │
│  │  用户界面(Shell/GUI)          │  │
│  │  系统调用接口(Syscall)        │  │
│  │  内核(进程/内存/文件/设备)    │  │
│  └───────────────────────────────┘  │
├─────────────────────────────────────┤
│            硬件 (CPU/内存/磁盘)     │
└─────────────────────────────────────┘
```

## 关系

**关键连接**：
- 操作系统 → **CPU**：进程调度，决定哪个程序使用CPU
- 操作系统 → **内存**：虚拟内存管理，进程隔离
- 操作系统 → **存储**：文件系统管理
- 操作系统 → **I/O设备**：驱动程序管理设备
- 应用程序 → **操作系统**：通过系统调用请求服务

## 操作系统职能

| 职能 | 说明 | 相关概念 |
|------|------|----------|
| 进程管理 | 创建、调度、终止进程 | 进程、线程、调度器 |
| 内存管理 | 分配/回收内存，虚拟内存 | 页面置换、内存分配器 |
| 文件系统 | 管理文件的存储和访问 | inode、目录、权限 |
| 设备管理 | 驱动程序，与硬件通信 | 中断、I/O、DMA |
| 安全与权限 | 用户认证、访问控制 | 用户、组、ACL |

## 内核类型

| 类型 | 特点 | 示例 |
|------|------|------|
| **宏内核 (Monolithic)** | 所有功能在内核中，通信快 | Linux, Unix |
| **微内核 (Microkernel)** | 只保留核心功能，其他在用户空间 | Minix, QNX |
| **混合内核** | 宏内核+微内核折中 | Windows NT, macOS XNU |
| **外内核 (Exokernel)** | 极简，库操作系统 | MIT ExOS |

## 常见操作系统

| OS | 内核 | 特点 | 应用场景 |
|----|------|------|----------|
| Linux | Linux | 开源、多平台 | 服务器、嵌入式、Android |
| Windows | Windows NT | 闭源、生态好 | PC、企业 |
| macOS | XNU (Mach+BSD) | Unix兼容 | Apple设备 |
| Android | Linux | 移动端霸主 | 手机、平板 |
| iOS | XNU | 闭源、严格权限 | iPhone/iPad |

## 系统调用 (System Call)

应用程序通过系统调用请求OS服务：

```c
// Linux 系统调用示例
#include <unistd.h>

int main() {
    // write 是系统调用
    const char* msg = "Hello, OS!\n";
    write(STDOUT_FILENO, msg, 13);  // sys_write

    // read 是系统调用
    char buf[100];
    read(STDIN_FILENO, buf, 100);   // sys_read

    // exit 也是系统调用
    _exit(0);                        // sys_exit
}
```

**系统调用 vs 函数调用**：

| 调用类型 | 执行位置 | 特权级别 |
|----------|----------|----------|
| 普通函数调用 | 用户空间 | 用户态 |
| 系统调用 | 内核空间 | 内核态 |

## 进程与程序

```python
# 进程是程序的运行实例
process = {
    "程序": "静态文件 (磁盘上的可执行文件)",
    "进程": "运行中的实例 (内存中的指令和数据)",
    "线程": "进程内的执行单元，共享进程资源"
}
```

```bash
# Linux 查看进程
ps aux           # 列出所有进程
top              # 实时查看系统状态
htop             # 更友好的交互式进程查看

# Windows
tasklist         # 列出进程
taskmgr          # 任务管理器
```

## 面试要点

1. **操作系统四大职能**：进程管理、内存管理、文件系统、设备管理
2. **宏内核 vs 微内核**：设计理念差异，Linux是宏内核
3. **系统调用**：应用程序与OS交互的唯一接口
4. **用户态 vs 内核态**：特权级别区别，为什么需要
5. **常见OS**：Linux、Windows、macOS、Android、iOS的内核特点
