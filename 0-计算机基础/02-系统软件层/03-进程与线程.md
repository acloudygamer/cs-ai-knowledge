# 进程与线程

## 概念

**进程**是程序的运行实例，拥有独立的地址空间。**线程**是进程内的执行单元，共享进程资源。

```
进程A                      进程B
┌─────────────────┐       ┌─────────────────┐
│ 线程1 (main)    │       │ 线程1 (main)    │
│  栈/寄存器/PC   │       │  栈/寄存器/PC   │
├─────────────────┤       ├─────────────────┤
│ 线程2           │       │                 │
│  栈/寄存器/PC   │       │                 │
├─────────────────┤       ├─────────────────┤
│  堆、代码、全局数据、文件描述符（线程共享）│       │  (独立资源)     │
└─────────────────┘       └─────────────────┘
```

## 关系

**关键连接**：
- 进程 → **CPU**：进程是CPU调度的基本单位
- 线程 → **进程**：线程属于进程，共享进程资源
- 线程 → **CPU**：线程是CPU调度的实际单位
- 进程 → **内存**：每个进程有独立的虚拟地址空间

## 进程

```bash
# Linux 查看进程
ps aux                    # 详细进程列表
ps -ef                    # 父子关系
pstree                    # 进程树

# 查看特定进程
cat /proc/<pid>/status   # 进程状态
ls -l /proc/<pid>/fd     # 打开的文件描述符
```

```python
# Python 多进程
from multiprocessing import Process

def worker(name):
    print(f"Worker {name} running")

if __name__ == "__main__":
    p = Process(target=worker, args=("A",))
    p.start()
    p.join()
```

### 进程状态

```
创建 → 就绪 ──调度──→ 运行 ──退出─→ 终止
         ↑       ↓
         └─阻塞─┘

就绪态: 等待CPU时间片
运行态: 正在执行指令
阻塞态: 等待I/O或其他资源
```

### 进程间通信 (IPC)

| 方式 | 特点 | 适用场景 |
|------|------|----------|
| 管道 | 半双工，亲缘进程 | 简单数据流 |
| 消息队列 | 消息结构化 | 异步通信 |
| 共享内存 | 最快，无拷贝 | 高性能 |
| 信号 | 异步通知 | 异常处理 |
| Socket | 跨主机通信 | 网络通信 |

```c
// 共享内存示例
#include <sys/shm.h>

int shmid = shmget(KEY, SIZE, IPC_CREAT | 0666);
void* addr = shmat(shmid, NULL, 0);
// 多个进程可同时访问addr
shmdt(addr);
shmctl(shmid, IPC_RMID, NULL);
```

## 线程

```python
# Python 多线程
import threading

def worker(name):
    print(f"Thread {name} running")

t = threading.Thread(target=worker, args=("A",))
t.start()
t.join()
```

### 线程 vs 进程

| 特性 | 进程 | 线程 |
|------|------|------|
| 资源 | 独立地址空间 | 共享进程资源 |
| 创建速度 | 慢 (需复制资源) | 快 (只需创建TCB) |
| 通信 | 需IPC | 直接读写共享内存 |
| 隔离性 | 强 | 弱（一个线程崩溃影响全进程） |
| 并行 | 多进程可跑满多核 | 多线程受GIL限制(Python) |

### GIL (全局解释器锁)

Python的多线程受GIL限制，同一时间只有一个线程执行Python字节码：

```python
# CPU密集型任务：多线程无法并行
import threading

def cpu_task():
    total = sum(i * i for i in range(10000000))

# 4个线程并不能加速CPU密集型任务
threads = [threading.Thread(target=cpu_task) for _ in range(4)]
```

**解决方案**：
- CPU密集型：用多进程 (multiprocessing)
- I/O密集型：多线程可行，或 asyncio

## 调度

### Linux CFS 调度器

完全公平调度器 (Completely Fair Scheduler)：
- 按权重分配CPU时间
- 虚拟运行时间越小，优先级越高

```bash
# 查看进程优先级
ps -eo pid,ni,cmd
# NI: nice值，-20到19，值越小优先级越高
```

### 协程 vs 线程 vs 进程

| 类型 | 创建成本 | 切换成本 | 并行性 |
|------|----------|----------|--------|
| 进程 | 高 | 高 | 可多核 |
| 线程 | 中 | 中 | 受GIL限制 |
| 协程 | 低 | 极低 | 单线程内并发 |

```python
# Python 协程 (asyncio)
import asyncio

async def worker():
    await asyncio.sleep(1)
    print("done")

asyncio.run(worker())
```

## 面试要点

1. **进程与线程区别**：资源、创建成本、通信方式
2. **进程状态转换**：就绪/运行/阻塞及转换条件
3. **IPC方式**：管道、消息队列、共享内存、socket对比
4. **Python GIL**：为什么存在，对多线程的影响
5. **进程调度**：CFS调度原理，nice值作用
6. **协程**：与线程的区别，asyncio的原理
