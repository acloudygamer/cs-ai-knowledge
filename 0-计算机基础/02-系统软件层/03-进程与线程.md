# 进程与线程

## 概念

**进程**是程序的运行实例，拥有独立的地址空间。**线程**是进程内的执行单元，共享进程资源。

```
进程A                      进程B
┌─────────────────┐       ┌─────────────────┐
│ 线程1 (main)    │       │ 线程1 (main)    │
│  栈/寄存器/PC   │       │  栈/寄存器/PC   │
├─────────────────┤       ├─────────────────┤
│ 线程2           │       │                 │
│  栈/寄存器/PC   │       │                 │
├─────────────────┤       ├─────────────────┤
│  堆、代码、全局数据、文件描述符（线程共享）│       │  (独立资源)     │
└─────────────────┘       └─────────────────┘
```

## 关系

**关键连接**：
- 进程 → **CPU**：进程是CPU调度的基本单位
- 线程 → **进程**：线程属于进程，共享进程资源
- 线程 → **CPU**：线程是CPU调度的实际单位
- 进程 → **内存**：每个进程有独立的虚拟地址空间

## 进程

```bash
# Linux 查看进程
ps aux                    # 详细进程列表
ps -ef                    # 父子关系
pstree                    # 进程树

# 查看特定进程
cat /proc/<pid>/status   # 进程状态
ls -l /proc/<pid>/fd     # 打开的文件描述符
```

```python
# Python 多进程
from multiprocessing import Process

def worker(name):
    print(f"Worker {name} running")

if __name__ == "__main__":
    p = Process(target=worker, args=("A",))
    p.start()
    p.join()
```

### 进程状态

```
创建 → 就绪 ──调度──→ 运行 ──退出─→ 终止
         ↑       ↓
         └─阻塞─┘

就绪态: 等待CPU时间片
运行态: 正在执行指令
阻塞态: 等待I/O或其他资源
```

### 进程间通信 (IPC)

| 方式 | 特点 | 适用场景 |
|------|------|----------|
| 管道 | 半双工，亲缘进程 | 简单数据流 |
| 消息队列 | 消息结构化 | 异步通信 |
| 共享内存 | 最快，无拷贝 | 高性能 |
| 信号 | 异步通知 | 异常处理 |
| Socket | 跨主机通信 | 网络通信 |

```c
// 共享内存示例
#include <sys/shm.h>

int shmid = shmget(KEY, SIZE, IPC_CREAT | 0666);
void* addr = shmat(shmid, NULL, 0);
// 多个进程可同时访问addr
shmdt(addr);
shmctl(shmid, IPC_RMID, NULL);
```

## 线程

```python
# Python 多线程
import threading

def worker(name):
    print(f"Thread {name} running")

t = threading.Thread(target=worker, args=("A",))
t.start()
t.join()
```

### 线程 vs 进程

| 特性 | 进程 | 线程 |
|------|------|------|
| 资源 | 独立地址空间 | 共享进程资源 |
| 创建速度 | 慢 (需复制资源) | 快 (只需创建TCB) |
| 通信 | 需IPC | 直接读写共享内存 |
| 隔离性 | 强 | 弱（一个线程崩溃影响全进程） |
| 并行 | 多进程可跑满多核 | 多线程受GIL限制(Python) |

### GIL (全局解释器锁)

Python的多线程受GIL限制，同一时间只有一个线程执行Python字节码：

```python
# CPU密集型任务：多线程无法并行
import threading

def cpu_task():
    total = sum(i * i for i in range(10000000))

# 4个线程并不能加速CPU密集型任务
threads = [threading.Thread(target=cpu_task) for _ in range(4)]
```

**解决方案**：
- CPU密集型：用多进程 (multiprocessing)
- I/O密集型：多线程可行，或 asyncio

## 调度

### Linux CFS 调度器

完全公平调度器 (Completely Fair Scheduler)：
- 按权重分配CPU时间
- 虚拟运行时间越小，优先级越高
- 红黑树组织就绪队列，O(log n) 调度

```bash
# 查看进程优先级
ps -eo pid,ni,cmd
# NI: nice值，-20到19，值越小优先级越高

# 设置nice值
renice -n 10 -p <pid>      # 降低优先级
nice -n -5 ./program       # 启动时设置高优先级
```

### 调度算法

| 算法 | 特点 | 适用场景 |
|------|------|----------|
| FCFS | 先来先服务，简单 | 批处理系统 |
| SJF | 最短作业优先 | 追求最小平均等待时间 |
| RR | 时间片轮转 | 分时系统，交互式任务 |
| CFS | 完全公平，Linux默认 | 通用服务器 |
| 实时调度 | 固定优先级 | 嵌入式/工业控制 |

```c
// Linux 实时调度 (需要root权限)
struct sched_param param = {.sched_priority = 50};
sched_setscheduler(pid, SCHED_FIFO, &param);
// SCHED_FIFO: 实时FIFO，无时间片
// SCHED_RR: 实时轮转，有时间片
// SCHED_NORMAL: 普通CFS调度
```

## 特殊进程状态

### 僵尸进程 (Zombie)

子进程退出后，父进程未调用wait()读取退出状态，此时子进程成为僵尸：

```bash
# 查看僵尸进程
ps aux | grep 'Z'
# 僵死进程标记为 Z，命令行显示为 <defunct>

# 僵尸进程处理
# 方法1: kill -SIGCHLD 父进程 (让它调用wait)
# 方法2: 杀死父进程，僵尸进程被init收养后清理
kill -9 <父进程pid>
```

```c
// 僵尸进程产生原因
pid_t pid = fork();
if (pid == 0) {
    // 子进程退出
    _exit(0);  // 此时变成僵尸，等待父进程wait()
} else {
    sleep(60);  // 父进程不调用wait，子进程保持僵尸
    wait(NULL); // 最终清理
}
```

### 孤儿进程 (Orphan)

父进程提前退出，子进程被init/systemd收养：

```bash
# 孤儿进程演示
# 父进程退出后，子进程PPID变为1(init)
ps -eo pid,ppid,cmd | grep 子进程名
```

## 进程生命周期

```
fork() → 就绪 → 运行 ──exit()──→ 僵尸 ──wait()──→ 终止
              ↑     │                  ↑
              └──阻塞┘                  │
                  (sleep/I/O)     父进程未wait
```

## 线程同步

多线程共享进程资源，需要同步机制防止竞态条件：

### 互斥锁 (Mutex)

```python
import threading

lock = threading.Lock()

def critical_section():
    lock.acquire()         # 获取锁
    try:
        # 访问共享资源
        shared_counter += 1
    finally:
        lock.release()     # 释放锁
```

### 信号量 (Semaphore)

```python
import threading

sem = threading.Semaphore(3)  # 最多3个并发

def limited_access():
    sem.acquire()
    try:
        # 使用受限资源
        pass
    finally:
        sem.release()
```

### 条件变量 (Condition)

```python
import threading

cond = threading.Condition()

def consumer():
    cond.acquire()
    while not item_available:
        cond.wait()       # 等待信号
    item = get_item()
    cond.release()

def producer():
    cond.acquire()
    put_item()
    cond.notify()         # 发送通知
    cond.release()
```

### 死锁 (Deadlock)

多个进程/线程互相等待对方持有的锁：

```python
# 死锁示例：两个线程互相等待对方的锁
import threading

lock1 = threading.Lock()
lock2 = threading.Lock()

def thread1():
    lock1.acquire()
    lock2.acquire()  # 等待thread2释放lock2
    lock2.release()
    lock1.release()

def thread2():
    lock2.acquire()
    lock1.acquire()  # 等待thread1释放lock1
    lock1.release()
    lock2.release()

# 避免死锁: 按固定顺序获取锁，或使用try_lock
```

**死锁必要条件（满足任一即可预防）**：
1. 互斥：资源不可共享
2. 持有并等待：持有资源同时请求更多
3. 不可抢占：资源不能被强制夺走
4. 循环等待：形成锁的环形依赖

## 线程池

预先创建一组线程，复用执行多个任务：

```python
from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(max_workers=4) as executor:
    # 提交任务
    future1 = executor.submit(task, arg1)
    future2 = executor.map(task, args_list)  # 批量提交

    # 获取结果
    result = future1.result()
```

```python
# 进程池（绕过GIL，适合CPU密集型）
from concurrent.futures import ProcessPoolExecutor

with ProcessPoolExecutor(max_workers=4) as executor:
    results = executor.map(cpu_intensive_func, data_list)
```

## 协程 vs 线程 vs 进程

| 类型 | 创建成本 | 切换成本 | 并行性 |
|------|----------|----------|--------|
| 进程 | 高 | 高 | 可多核 |
| 线程 | 中 | 中 | 受GIL限制 |
| 协程 | 低 | 极低 | 单线程内并发 |

```python
# Python 协程 (asyncio)
import asyncio

async def worker():
    await asyncio.sleep(1)
    print("done")

async def main():
    # 并发执行多个协程
    await asyncio.gather(
        worker(),
        worker(),
        worker()
    )

asyncio.run(main())
```

### asyncio 原理

```
事件循环(Event Loop)
    ↓
检查就绪的协程
    ↓
执行协程直到 await → 挂起
    ↓
等待I/O完成回调
    ↓
恢复挂起的协程
```

asyncio基于生成器(generator)实现协程，await时让出控制权给事件循环，非阻塞等待I/O。
