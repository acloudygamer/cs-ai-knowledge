# 解释器

## 概念

**解释器**逐行读取源代码，直接执行对应操作，不需要预先编译成机器码。

```
源代码 → [解释器] → 逐行执行 → 输出结果
```

## 关系

**关键连接**：
- 解释器 → **运行时**：执行时依赖运行时提供环境（内存管理、库函数）
- 解释器 → **操作系统**：通过系统调用与OS交互
- 解释器 ← **编译器**：有些语言先编译后解释（如Java的javac编译成字节码，JVM解释执行）

## 解释执行 vs 编译执行

| 特性 | 解释执行 | 编译执行 |
|------|----------|----------|
| 执行速度 | 慢（每次都要翻译） | 快 |
| 启动速度 | 快（不需要编译） | 慢 |
| 跨平台 | 源代码跨平台（如有解释器） | 需要针对平台编译 |
| 调试 | 容易（可直接修改） | 需要重新编译 |

## Python 解释器

Python 是典型的解释型语言，但有多种实现：

```bash
# CPython：官方实现，C语言编写
python script.py

# PyPy：JIT 编译，速度更快
pypy script.py

# Jython：运行在 JVM 上
jython script.py

# 查看 Python 解释器路径
which python
python --version
```

## REPL

**Read-Eval-Print Loop**：交互式解释器环境

```python
# Python REPL
$ python
>>> 1 + 1
2
>>> for i in range(3):
...     print(i)
...
0
1
2
```

```javascript
// Node.js REPL
$ node
> 1 + 1
2
> .help
.break    .clear    .exit    .help    .load    .save
```

## 字节码解释

有些语言（如 Python、Java）将源代码先编译成**字节码**，再由虚拟机解释执行：

```
源代码 → 编译器 → 字节码 → 虚拟机(解释执行)
                              ↓
                         JIT 编译(热点代码优化)
```

**为什么需要字节码？**
- 编译一次，多次执行（比纯解释快）
- 跨平台：只要有对应平台的虚拟机即可运行

## 面试要点

1. **解释执行 vs 编译执行**：速度、启动、跨平台对比
2. **Python 多种实现**：CPython、PyPy、Jython 区别
3. **REPL 用途**：快速测试、调试、学习
4. **字节码作用**：平衡编译与解释的执行效率
