# 运行时

## 概念

**运行时**是程序执行时依赖的环境，包括内存管理、垃圾回收、线程调度、库函数等。

```
源代码 → 编译器/解释器 → 可执行文件/字节码 → [运行时] → 操作系统 → 硬件
```

## 关系

**关键连接**：
- 运行时 → **操作系统**：通过系统调用请求资源
- 运行时 → **内存管理**：分配/释放内存，垃圾回收
- 运行时 → **线程调度**：管理并发执行的线程
- 编译器 → 运行时：编译时决定生成什么类型的运行时依赖

## 各语言的运行时

| 语言 | 运行时 | 特点 |
|------|--------|------|
| C | C RunTime (CRT) | 最小化，主要初始化代码 |
| C++ | 无独立运行时（依赖CRT） | 通常编译进可执行文件 |
| Java | JVM (Java Runtime) | 包含GC、JIT、类加载 |
| Go | Go Runtime | 协程调度、GC、栈管理 |
| Python | Python Runtime | 解释器内置，动态类型 |
| Rust | 几乎无运行时 | 内存管理在编译时确定 |

## 内存管理

运行时负责**堆内存分配和回收**：

```python
# Python：自动垃圾回收
x = [1, 2, 3]    # 堆上分配
del x             # 标记为可回收
import gc
gc.collect()      # 手动触发GC
```

```go
// Go：并发GC，STW时间短
var slice []int = make([]int, 100)  // 堆上分配
```

## 垃圾回收 (GC)

**常见GC算法**：

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 引用计数 | 记录引用数，为0时回收 | 及时回收 | 无法处理循环引用 |
| 标记-清除 | 标记可达对象，清除不可达 | 处理循环引用 | STW停顿 |
| 分代收集 | 新对象频繁回收，老对象少回收 | 效率高 | 需要调整参数 |

## 栈 vs 堆

| 区域 | 用途 | 分配方式 | 大小 |
|------|------|----------|------|
| 栈 | 函数调用、局部变量 | 编译器自动管理 | 较小（1-8MB） |
| 堆 | 动态分配内存 | 运行时手动/自动 | 大（受物理内存限制） |

```python
# 栈：函数调用
def foo():
    x = 10        # 栈上分配
    bar(x)        # 调用结束，x自动释放

# 堆：动态分配
data = [1, 2, 3] # 堆上分配，data是栈上的指针
```

## 运行时库

提供程序运行所需的基础库函数：

- **libc** (Linux)：C标准库，提供malloc、printf等
- **glibc** (GNU C Library)：Linux默认C库
- **MSVCRT** (Windows)：Microsoft C Run-Time

## 面试要点

1. **运行时 vs 编译器**：各自负责什么
2. **GC算法**：引用计数、标记-清除、分代收集的原理和优缺点
3. **栈 vs 堆**：区别、使用场景、内存管理方式
4. **各语言运行时对比**：为什么Java有独立运行时，Go的runtime特点
