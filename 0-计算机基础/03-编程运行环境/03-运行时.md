# 运行时

## 概念

**运行时**是程序执行时依赖的环境，包括内存管理、垃圾回收、线程调度、库函数等。运行时在程序启动时加载，在程序结束时释放，是程序与操作系统之间的桥梁。

```
源代码 → 编译器/解释器 → 可执行文件/字节码 → [运行时] → 操作系统 → 硬件
```

## 关系

**关键连接**：
- 运行时 → **操作系统**：通过系统调用请求资源（内存、文件、网络）
- 运行时 → **内存管理**：分配/释放堆内存，执行垃圾回收
- 运行时 → **线程调度**：管理协程/线程的创建、切换、销毁
- 编译器 → 运行时：编译时决定生成什么类型的运行时依赖
- 运行时 → **JIT编译器**：热点代码的运行时编译（Java、JavaScript、V8）

## 各语言的运行时

| 语言 | 运行时 | 特点 |
|------|--------|------|
| C | C RunTime (CRT) | 最小化，主要初始化代码，malloc/free 手动管理 |
| C++ | 无独立运行时（依赖CRT） | 通常编译进可执行文件，RAII 自动管理 |
| Java | JVM (Java Runtime) | 包含GC、JIT、类加载，多线程并发 |
| Go | Go Runtime | 协程调度（GMP模型）、并发GC、栈管理 |
| Python | Python Runtime | 解释器内置，GIL限制，动态类型 |
| JavaScript | JS Engine (V8/SpiderMonkey) | JIT编译、GC、事件循环 |
| Rust | 几乎无运行时 | 内存管理在编译时通过 ownership 确定 |

## 内存管理

运行时负责**堆内存分配和回收**：

```python
# Python：自动垃圾回收
x = [1, 2, 3]    # 堆上分配对象
y = x            # 引用计数 +1
del x             # 引用计数 -1，x 不再引用该对象
import gc
gc.collect()      # 手动触发GC（通常不需要）
```

```go
// Go：堆上分配，GC 自动回收
var slice []int = make([]int, 100)  // 堆上分配
ptr := new(int)                     // 堆上分配
```

```c
// C：手动分配和释放
int *p = malloc(sizeof(int) * 10);  // 堆上分配
free(p);                              // 手动释放
p = NULL;                             // 避免悬空指针
```

## 垃圾回收 (GC)

**常见GC算法**：

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 引用计数 | 每个对象记录引用数量，为0时立即回收 | 及时回收、无STW | 无法处理循环引用、计数更新开销 |
| 标记-清除 | 标记所有可达对象，清除不可达对象 | 处理循环引用 | STW停顿（Stop-The-World） |
| 分代收集 | 新对象放年轻代（频繁回收）、老对象放老年代（少回收） | 效率高，符合对象生命周期 | 需要调整参数，内存碎片 |
| 标记-整理 | 标记后移动存活对象，整理内存 | 无碎片 | 移动开销 |

### G1GC（Java 默认）

将堆划分为多个 Region，优先回收垃圾最多的 Region：

```bash
# JVM G1GC 调优参数
java -XX:+UseG1GC -XX:MaxGCPauseMillis=200 -jar app.jar
```

## 栈 vs 堆

| 区域 | 用途 | 分配方式 | 大小 | 速度 |
|------|------|----------|------|------|
| 栈 | 函数调用、局部变量、返回地址 | 编译器自动管理（压栈/弹栈） | 较小（1-8MB） | 快 |
| 堆 | 动态分配内存、对象 | 运行时手动（malloc/new）或自动（GC） | 大（受物理内存限制） | 慢 |

```python
# 栈：函数调用
def foo():
    x = 10        # 栈上分配（整数是不可变对象，x 是引用）
    bar(x)        # 调用结束，x 栈帧弹出

# 堆：动态分配
data = [1, 2, 3]  # 堆上分配列表对象，data 是栈上的引用
```

```c
// 栈 vs 堆 对比
void example() {
    int local_var = 5;      // 栈：函数返回时自动释放
    int *heap_var = malloc(sizeof(int));  // 堆：需要手动 free
    *heap_var = 10;
    free(heap_var);         // 忘记会导致内存泄漏
}
```

## 运行时库

提供程序运行所需的基础库函数：

- **libc** (Linux)：C标准库，提供malloc、printf、open等
- **glibc** (GNU C Library)：Linux 默认 C 库，POSIX 兼容
- **MSVCRT** (Windows)：Microsoft C Run-Time
- **libstdc++** (C++)：C++ 标准库容器、算法

## 线程 vs 协程

| 模型 | 调度 | 切换成本 | 适用场景 |
|------|------|----------|----------|
| 线程（内核级） | OS 调度，抢占式 | 大（KB级栈 + 上下文切换） | CPU 密集型、IO 阻塞 |
| 协程（用户级） | 用户代码调度协作式 | 小（栈帧切换） | 高并发 IO、异步编程 |

```go
// Go 协程示例
func main() {
    go func() {  // 轻量级协程，栈初始 2KB
        fmt.Println("并发执行")
    }()
    time.Sleep(time.Second)
}
```
