# 虚拟机

## 概念

**虚拟机**是通过软件模拟的计算机系统，可以运行操作系统的完整实例，或运行特定语言的字节码。

```
物理机 → [虚拟机管理器(Hypervisor)] → 多个虚拟机 → 各自运行操作系统
物理机 → [语言虚拟机] → 字节码程序
```

## 关系

**关键连接**：
- 虚拟机 → **操作系统**：系统级虚拟机需要宿主OS或直接运行在硬件上
- 虚拟机 → **硬件**：虚拟化硬件资源（CPU、内存、磁盘）
- 虚拟机 ← **编译器**：编译器生成字节码，虚拟机解释/编译执行

## 两类虚拟机

### 1. 系统级虚拟机

模拟完整计算机，運行操作系统：

| 虚拟机 | 特点 | 用途 |
|--------|------|------|
| VMware | 商业级，性能好 | Windows/Mac运行Linux |
| VirtualBox | 开源，跨平台 | 开发测试环境 |
| KVM | Linux内核模块 | 服务器虚拟化 |
| Docker | 容器级，轻量 | 应用的隔离打包 |

```bash
# Docker vs 虚拟机
虚拟机：完整OS + 虚拟硬件 → 启动慢（分钟级），资源占用大
容器：共享宿主机内核 → 启动快（秒级），资源占用小
```

### 2. 语言虚拟机

运行特定语言的字节码：

| 虚拟机 | 语言 | 特点 |
|--------|------|------|
| JVM | Java, Kotlin, Scala | JIT编译，跨平台 |
| CLR | C#, F# | .NET平台 |
| BEAM | Elixir, Erlang | 并发模型强大 |
| BPF VM | Linux内核 | 网络/跟踪程序 |

## JVM 详解

Java虚拟机是**语言虚拟机**的代表：

```
.java文件 → javac编译 → .class字节码 → JVM解释/JIT编译 → 机器码
```

**JVM 核心组件**：

```
┌─────────────────────────────────────┐
│              JVM                    │
├─────────────────────────────────────┤
│  类加载器 (ClassLoader)             │
│  运行时数据区 (Heap, Stack, etc.)   │
│  执行引擎                           │
│    ├─ 解释器                        │
│    └─ JIT编译器                     │
│  本地接口 (JNI)                     │
└─────────────────────────────────────┘
```

**JIT 编译**：热点代码（频繁执行的）会被即时编译成机器码，大幅提升性能

```java
// JIT监控热点代码，编译后执行
for (int i = 0; i < 1000000; i++) {
    sum += i;  // 这段代码可能被JIT编译
}
```

## Docker 容器

容器不是传统虚拟机，但提供了类似的隔离能力：

```bash
# Dockerfile 示例
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "app.py"]

# 构建和运行
docker build -t myapp .
docker run -p 8000:8000 myapp
```

**容器 vs 虚拟机**：

| 特性 | 容器 | 虚拟机 |
|------|------|--------|
| 启动时间 | 秒级 | 分钟级 |
| 资源占用 | 小（MB级） | 大（GB级） |
| 隔离性 | 进程级 | 完整OS |
| 性能 | 几乎无损耗 | 有损耗 |

## 面试要点

1. **系统虚拟机 vs 语言虚拟机**：目的和实现差异
2. **Docker vs 虚拟机**：原理区别、使用场景
3. **JVM组成**：类加载器、运行时数据区、执行引擎
4. **JIT编译**：热点代码优化的原理
5. **容器隔离原理**：namespace、cgroup
