# 虚拟机

## 概念

**虚拟机**是通过软件模拟的计算机系统，可以运行操作系统的完整实例，或运行特定语言的字节码。虚拟机将物理资源虚拟化，提供隔离的运行环境。

```
物理机 → [虚拟机管理器(Hypervisor)] → 多个虚拟机 → 各自运行操作系统
物理机 → [语言虚拟机] → 字节码程序
```

## 关系

**关键连接**：
- 虚拟机 → **操作系统**：系统级虚拟机需要宿主OS（Type 2）或直接运行在硬件上（Type 1）
- 虚拟机 → **硬件**：虚拟化硬件资源（CPU、内存、磁盘、网络）
- 虚拟机 ← **编译器**：编译器生成字节码，虚拟机解释/JIT执行
- 容器 → **Linux内核**：共享宿主机内核，通过 namespace 和 cgroup 隔离

## 两类虚拟机

### 1. 系统级虚拟机（Hypervisor）

模拟完整计算机，运行操作系统：

| 类型 | 代表 | 特点 | 运行环境 |
|------|------|------|----------|
| Type 1 (裸金属) | VMware ESXi, KVM, Hyper-V | 直接运行在硬件上 | 服务器、企业 |
| Type 2 (宿主) | VirtualBox, VMware Workstation | 运行在宿主机OS上 | 桌面、开发 |

| 虚拟机 | 特点 | 用途 |
|--------|------|------|
| VMware | 商业级，性能好 | Windows/Mac运行Linux |
| VirtualBox | 开源，跨平台 | 开发测试环境 |
| KVM | Linux内核模块，本质是Type 1 | 服务器虚拟化 |
| Hyper-V | Windows内置 | Windows服务器 |

### 2. 语言虚拟机

运行特定语言的字节码：

| 虚拟机 | 语言 | 特点 |
|--------|------|------|
| JVM | Java, Kotlin, Scala | JIT编译，跨平台（Write Once Run Anywhere） |
| CLR | C#, F#, VB.NET | .NET平台，托管代码 |
| BEAM | Elixir, Erlang | Actor模型，并发强大 |
| BPF VM | Linux内核 | 网络/跟踪程序，安全沙箱 |
| WASM Runtime | WebAssembly | 浏览器/服务端沙箱执行 |

## Docker 容器

容器不是传统虚拟机，通过**Linux namespace**和**cgroup**实现进程级隔离：

```bash
# Docker vs 虚拟机 原理对比
虚拟机：完整OS内核 + 虚拟硬件 → 启动慢（分钟级），资源占用大（GB级）
容器：共享宿主机内核 + namespace隔离 → 启动快（秒级），资源占用小（MB级）
```

### 容器隔离技术

```bash
# namespace 隔离：PID、网络、挂载、UTS、IPC、User
# cgroup 限制：CPU、内存、IO、进程数

# 查看容器的 namespace
ls -la /proc/self/ns/

# 查看 cgroup 限制
cat /sys/fs/cgroup/memory/memory.limit_in_bytes
```

```bash
# Dockerfile 示例
FROM python:3.11-slim           # 基础镜像（层叠复用）
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt  # 构建时执行，构建缓存
COPY . .                              # 复制应用代码
EXPOSE 8000                           # 声明端口
CMD ["python", "app.py"]              # 启动命令

# 构建和运行
docker build -t myapp .               # -t 指定镜像名
docker run -p 8000:8000 --memory=512m myapp  # -p 端口映射，--memory 限制内存
```

**容器 vs 虚拟机**：

| 特性 | 容器 | 虚拟机 |
|------|------|--------|
| 启动时间 | 秒级 | 分钟级 |
| 资源占用 | 小（MB级） | 大（GB级） |
| 隔离性 | 进程级 | 完整OS |
| 性能 | 几乎无损耗 | 有损耗（虚拟化层） |
| 数量密度 | 高（数十/百容器/主机） | 低（数虚拟机/主机） |
| 适用场景 | 微服务、CI/CD、Serverless | 多系统、安全隔离 |

## JVM 详解

Java虚拟机是**语言虚拟机**的代表，实现了"一次编写，到处运行"：

```
.java文件 → javac编译 → .class字节码 → JVM解释/JIT编译 → 机器码
```

**JVM 核心组件**：

```
┌─────────────────────────────────────┐
│              JVM                    │
├─────────────────────────────────────┤
│  类加载器 (ClassLoader)             │
│    ├─ Bootstrap ClassLoader (rt.jar) │
│    ├─ Extension ClassLoader         │
│    └─ Application ClassLoader       │
├─────────────────────────────────────┤
│  运行时数据区                       │
│    ├─ Heap（对象分配、GC）           │
│    ├─ Method Area（类信息）          │
│    ├─ Stack（线程栈）               │
│    ├─ PC Register（指令地址）        │
│    └─ Native Stack                  │
├─────────────────────────────────────┤
│  执行引擎                           │
│    ├─ 解释器（字节码→机器码）        │
│    ├─ JIT编译器（热点代码优化）      │
│    └─ GC（垃圾回收）                 │
├─────────────────────────────────────┤
│  本地接口 (JNI)                     │
└─────────────────────────────────────┘
```

**JIT 编译**：热点代码（频繁执行的）会被即时编译成机器码，大幅提升性能

```java
// JIT监控热点代码，编译后执行
// 解释执行 + JIT 编译混合模式
for (int i = 0; i < 1000000; i++) {
    sum += i;  // 这段代码可能被JIT编译成本地机器码
}

// JIT 编译阈值可通过 -XX:CompileThreshold 调整
// 热点检测：方法调用计数 + 循环回边计数
```

### 类加载过程

public class MyClass {

## BEAM（Erlang/Elixir 虚拟机）

BEAM 是 Erlang 的虚拟机，以** Actor 模型**和**轻量级进程**著称：

```erlang
% Erlang 示例：10万个并发进程
spawn(fun() -> loop(0) end).
% 每个进程 300+ 字节省栈，进程间通过消息传递通信
% 进程故障隔离（let it crash）
```

**BEAM vs JVM**：
- BEAM：协作式调度（每个进程必须主动让出控制权）
- JVM：抢占式调度（OS 调度线程）
- BEAM 更适合电信/金融等需要强一致性的场景
