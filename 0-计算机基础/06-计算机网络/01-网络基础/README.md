# 网络分层模型

## 概念

计算机网络通过分层架构管理复杂性，每层负责特定功能，层间通过接口通信。

```
OSI七层模型                          TCP/IP四层模型
┌─────────────────────────────────┐  ┌───────────────────────┐
│ 7 应用层 (Application)          │  │                       │
├─────────────────────────────────┤  │  应用层               │
│ 6 表示层 (Presentation)         │  │                       │
├─────────────────────────────────┤  ├───────────────────────┤
│ 5 会话层 (Session)              │  │                       │
├─────────────────────────────────┤  │  传输层               │
│ 4 传输层 (Transport)      TCP/UDP │  ├───────────────────────┤
├─────────────────────────────────┤  │                       │
│ 3 网络层 (Network)        IP     │  │  网络层               │
├─────────────────────────────────┤  ├───────────────────────┤
│ 2 数据链路层 (Data Link)  Ethernet│ │  网络接口层           │
├─────────────────────────────────┤  │                       │
│ 1 物理层 (Physical)             │  └───────────────────────┘
└─────────────────────────────────┘
```

## 关系

**关键连接**：
- 传输层 → **网络层**：TCP/UDP在IP协议之上传输数据
- DNS → **传输层**：DNS查询通常使用UDP端口53
- HTTP → **传输层**：HTTP/1.1和HTTP/2基于TCP，HTTP/3基于QUIC(UDP)
- 套接字 → **传输层**：应用程序通过Socket API使用TCP/UDP

## 各层职责

| 层级 | 协议 | 核心功能 | 数据单位 |
|------|------|---------|---------|
| 应用层 | HTTP, DNS, SMTP, FTP | 应用程序协议 | 消息/流 |
| 传输层 | TCP, UDP | 端到端可靠传输 | 段(TCP)/数据报(UDP) |
| 网络层 | IP, ICMP, BGP | 路由与寻址 | 包(Packet) |
| 数据链路层 | Ethernet, Wi-Fi, ARP | 局域网内传输 | 帧(Frame) |
| 物理层 | 光纤/铜缆/无线电 | 比特流传输 | 比特 |

## 协议数据单元 (PDU)

```
应用层:     消息 (HTTP请求/响应)
传输层:     段 (TCP Segment) / 数据报 (UDP Datagram)
网络层:     包 (IP Packet)
数据链路层: 帧 (Ethernet Frame)
物理层:     比特 (Bits)
```

## 常用端口

| 端口 | 协议 | 用途 |
|------|------|------|
| 80   | HTTP | Web流量 |
| 443  | HTTPS | 加密Web流量 |
| 53   | DNS | 域名解析 |
| 22   | SSH | 远程登录 |
| 25   | SMTP | 邮件发送 |
| 3306 | MySQL | 数据库连接 |
| 5432 | PostgreSQL | 数据库连接 |
| 6379 | Redis | 缓存服务 |

```bash
# 查看端口占用 (Linux)
ss -tulnp              # TCP/UDP监听端口
netstat -tulnp         # 较老的工具

# 查看特定端口
ss -tlnp | grep :80

# 查看网络连接状态
ss -s                 # 连接统计
```

## IP地址

```bash
# 查看IP地址 (Linux)
ip addr show
ifconfig               # 较老的命令

# Windows
ipconfig
ipconfig /all         # 详细信息
```

**IPv4 vs IPv6**：
| 特性 | IPv4 | IPv6 |
|------|------|------|
| 地址长度 | 32位 | 128位 |
| 地址格式 | 192.168.1.1 | 2001:0db8::1 |
| 地址数量 | ~40亿 | 无限 |
| 头部大小 | 可变 | 固定40字节 |

```bash
# IPv4子网掩码计算
# 192.168.1.0/24 表示网络前缀24位，主机位8位
# 可用主机数: 2^8 - 2 = 254 (排除网络地址和广播地址)

# 私有地址范围
# 10.0.0.0/8      (10.x.x.x)
# 172.16.0.0/12   (172.16.x.x - 172.31.x.x)
# 192.168.0.0/16  (192.168.x.x)
```

## 网络诊断

```bash
# 连通性测试
ping baidu.com                    # ICMP回显
ping6 ipv6.google.com             # IPv6

# 路由追踪
traceroute baidu.com              # Linux，看数据包经过的路由
tracert baidu.com                 # Windows

# DNS查询
nslookup baidu.com                # 基础DNS查询
dig baidu.com                     # 详细DNS信息
dig @8.8.8.8 baidu.com            # 指定DNS服务器

# 查看网络路径
curl -I https://baidu.com         # HTTP头部
mtr baidu.com                     # 持续路由+ping
```
