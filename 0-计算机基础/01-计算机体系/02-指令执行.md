# 指令执行

## 概念

CPU执行指令是**存储程序**概念的核心——程序以指令形式存储在内存中，CPU逐条取指、执行。

```
内存: [指令1][指令2][指令3]...
           ↓
        CPU: 取指 → 译码 → 执行 → 访存 → 写回
```

## 关系

**关键连接**：
- 指令执行 → **CPU**：CPU是执行指令的硬件
- 指令执行 → **内存**：指令和数据都存储在内存
- 编译器 → **指令集**：编译器生成特定CPU指令集的机器码
- 操作系统 → **指令执行**：OS管理进程调度，就是管理指令执行

**指令执行与性能**：
```
代码 → 编译器 → 汇编 → 机器码 → CPU执行
  │                              │
  └──── 优化级别 (O0/O1/O2/O3) ───┘
```
- `-O0`：无优化，调试友好
- `-O1`：基础优化，编译时间适中
- `-O2`：较高优化，推荐用于发布
- `-O3`：最高优化，可能增加代码体积

**流水线 (Pipeline)**：现代CPU并行执行多条指令
```
周期1: [取指1] [译码2] [执行3]
周期2: [取指2] [译码3] [执行4]
周期3: [取指3] [译码4] [执行5]
```

## 指令执行周期

CPU执行一条指令的完整周期：

```
┌─────────────┐
│   取指(Fetch)  │ ← 从内存获取指令
└──────┬──────┘
       ▼
┌─────────────┐
│   译码(Decode) │ ← 解析指令操作码和操作数
└──────┬──────┘
       ▼
┌─────────────┐
│   执行(Execute) │ ← ALU算术逻辑运算
└──────┬──────┘
       ▼
┌─────────────┐
│   访存(Memory) │ ← 访问内存（仅必要时）
└──────┬──────┘
       ▼
┌─────────────┐
│   写回(WriteBack)│ ← 结果写回寄存器
└─────────────┘
```

## 指令类型

| 指令类型 | 示例 | 说明 |
|----------|------|------|
| 算术指令 | ADD, SUB, MUL, DIV | 加减乘除 |
| 逻辑指令 | AND, OR, XOR, NOT | 逻辑运算 |
| 数据传输 | MOV, LOAD, STORE | 寄存器/内存间传输 |
| 控制流 | JMP, JZ, CALL, RET | 跳转、调用、返回 |
| I/O指令 | IN, OUT | 与外设交互 |

## 汇编语言示例

```asm
; x86 汇编示例：计算 1 + 2
section .text
global _start

_start:
    mov eax, 1      ; 将1加载到EAX寄存器
    mov ebx, 2      ; 将2加载到EBX寄存器
    add eax, ebx    ; EAX = EAX + EBX = 3

    ; 退出程序 (Linux syscall)
    mov eax, 1      ; syscall number: exit
    int 0x80        ; 触发软中断，进入内核
```

```python
# Python 编译成的字节码（简化）
import dis
dis.dis(lambda x: x + 1)

# 输出类似：
# LOAD_FAST    # 加载局部变量
# BINARY_ADD   # 加法操作
# RETURN_VALUE  # 返回结果
```

## 指令集架构 (ISA)

CPU指令集定义了软件与硬件的接口：

| 架构 | 特点 | 应用 |
|------|------|------|
| x86/x64 | 复杂指令集(CISC)，性能强 | PC/服务器（Intel/AMD） |
| ARM | 精简指令集(RISC)，功耗低 | 移动端、嵌入式 |
| RISC-V | 开源精简指令集 | 学术、物联网 |
| MIPS | 经典RISC | 路由器、嵌入式 |

```bash
# 查看CPU架构 (Linux)
cat /proc/cpuinfo | grep "model name"
arch  # 或 uname -m

# Windows
wmic cpu get name
```

## 编译执行全过程

```
源代码 (.c) → 预处理 → 编译 → 汇编 → 链接 → 可执行文件
     ↓
  加载到内存
     ↓
  CPU取指执行
     ↓
  操作系统调度
```

```c
// 源码
int add(int a, int b) {
    return a + b;
}

// 编译后的汇编 (简化)
add:
    push ebp
    mov ebp, esp
    mov eax, [ebp+8]    ; 取出参数a
    add eax, [ebp+12]   ; 加上参数b
    pop ebp
    ret
```

## 流水线 (Pipeline)

现代CPU通过流水线技术并行执行多条指令：

```python
# 五级流水线示意
pipeline_stages = {
    "IF": "取指 (Instruction Fetch)",
    "ID": "译码 (Instruction Decode)",
    "EX": "执行 (Execute)",
    "MEM": "访存 (Memory Access)",
    "WB": "写回 (Write Back)"
}

# 理想情况下每个周期完成一条指令
# 吞吐率 = 1条指令/周期 (vs 非流水线的1/5)
```

**流水线冒险 (Hazards)**：
| 类型 | 原因 | 解决方案 |
|------|------|---------|
| 结构冒险 | 硬件资源冲突 | 增加硬件资源 |
| 数据冒险 | 指令间数据依赖 | 转发/停顿 |
| 控制冒险 | 分支指令改变PC | 分支预测/延迟分支 |

```asm
; 数据冒险示例
mov eax, [mem1]    ; eax = mem1
add eax, ebx       ; 需要等上面的eax
mov [mem2], eax    ; 数据冒险
```

**分支预测**：
- 静态预测：总是预测分支不发生
- 动态预测：根据历史记录预测（2位饱和计数器）
- Intel的Inteligator：结合多种策略

## 进程与指令执行

操作系统通过**调度**决定哪个进程的指令在何时执行：

```python
# 进程状态转换（简化）
process_states = {
    "创建态": "进程刚被创建",
    "就绪态": "等待CPU时间片",
    "运行态": "正在执行指令",
    "阻塞态": "等待I/O完成",
    "终止态": "执行完毕，资源回收"
}
```

**时间片轮转**：OS给每个进程分配CPU时间片，轮流执行，看起来像是并行
