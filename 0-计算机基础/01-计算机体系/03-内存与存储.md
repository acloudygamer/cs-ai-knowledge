# 内存与存储

## 概念

**内存**是临时存储，用于运行时数据访问；**存储**是永久存储，用于持久化数据。

```
┌──────────────────────────────────────────┐
│              存储金字塔                    │
├──────────────────────────────────────────┤
│  CPU寄存器    速度最快，容量极小    0-cycle│
│  L1/L2/L3缓存                          │ ← CPU核心附近
├──────────────────────────────────────────┤
│  主存(RAM)    速度快，容量GB级      100+  │
├──────────────────────────────────────────┤
│  SSD/磁盘     速度慢，容量TB级      10万+ │
├──────────────────────────────────────────┤
│  网络存储/云   速度最慢，容量无限         │
└──────────────────────────────────────────┘
```

## 关系

**关键连接**：
- 内存 ↔ **CPU**：CPU直接读写内存，延迟低
- 存储 → **内存**：程序启动时从存储加载到内存
- 内存 → **存储**：数据需要持久化时写入存储
- 虚拟内存 → **物理内存**：操作系统通过虚拟内存管理物理内存

**内存墙 (Memory Wall)**：
```
CPU性能提升 (每年~50%)
    vs
内存带宽提升 (每年~10%)
    ↓
差距越来越大 → 缓存变得越来越重要
```

**局部性原理 (Locality)**：
- **时间局部性**：访问过的数据近期可能再次访问
- **空间局部性**：访问的数据附近的数据可能也被访问
```python
# 时间局部性示例
for i in range(1000):
    result = cache[i % 10]  # 反复访问同一数据
# 空间局部性示例
for i in range(1000):
    process(array[i])  # 顺序访问邻接数据
```

## 内存层次结构

### 寄存器 (Registers)
```python
# CPU寄存器是最快的存储
registers = {
    "PC/IP": "程序计数器，指向下一条指令",
    "SP": "栈指针，指向当前栈顶",
    "BP": "基址指针，用于函数调用",
    "AX/BX/CX/DX": "通用寄存器",
    "FLAGS": "状态标志位"
}
```

### 缓存 (Cache)
```python
# 多级缓存减少CPU访问内存的延迟
cache_levels = {
    "L1": {"size": "32-64KB", "latency": "4 cycles"},
    "L2": {"size": "256KB-1MB", "latency": "12 cycles"},
    "L3": {"size": "4-64MB", "latency": "40 cycles"}
}
```

**缓存映射方式**：
| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 直接映射 | 每个块对应唯一缓存位置 | 简单，成本低 | 冲突多 |
| 组相联 | 每组多个缓存位置 | 平衡性能与成本 | 复杂度增加 |
| 全相联 | 任何位置都可以放 | 最灵活 | 硬件成本高 |

**缓存行 (Cache Line)**：
```python
# 典型缓存行大小: 64字节
cache_line = {
    "大小": "64 bytes",
    "Tag": "用于匹配主存地址",
    "Valid Bit": "是否有效",
    "Dirty Bit": "是否被修改",
}
```

**缓存写策略**：
- **Write-Through**：同时写缓存和内存，简单但慢
- **Write-Back**：只写缓存，替换时再写内存，复杂但快
```python
write_through = "每次写都写内存，保证一致"
write_back = "延迟写内存，减少内存带宽"
```

### 主存 (RAM)
```c
// C语言内存区域
int global = 10;                    // .data段（全局变量）
static int static_var = 20;        // .data段（静态变量）

int main() {
    int local = 30;                // 栈（局部变量）
    int* heap = malloc(sizeof(int)); // 堆（动态分配）
    free(heap);
    return 0;
}
```

## 虚拟内存

虚拟内存让程序以为自己拥有连续的内存空间：

```
进程A的虚拟地址空间          物理内存
┌────────────────┐         ┌────────┐
│ 0x0000 - 0xFFFF│ ←映射→  │ 物理页 │
└────────────────┘         └────────┘
```

**优点**：
1. 进程隔离，互不干扰
2. 简化内存管理
3. 支持物理内存扩展

```bash
# Linux 查看虚拟内存信息
cat /proc/meminfo
vmstat 1

# Windows
systeminfo | findstr /C:"Virtual Memory"
```

## 内存分页

虚拟内存按**页**管理，典型大小4KB：

```python
# 内存分页示意
page_table = {
    "进程A": {
        "虚拟页0": "物理页3",
        "虚拟页1": "物理页7",
        "虚拟页2": "未分配（缺页）"
    }
}
```

**缺页异常 (Page Fault)**：访问未映射的页时，OS介入加载对应数据

## 栈 vs 堆

| 特性 | 栈 (Stack) | 堆 (Heap) |
|------|------------|-----------|
| 管理方式 | 编译器自动管理 | 程序员手动管理 |
| 空间大小 | 较小（1-8MB） | 受物理内存限制 |
| 分配速度 | 快 | 慢 |
| 释放方式 | 函数返回自动释放 | free/delete手动释放 |

```c
// 栈 vs 堆示例
void function() {
    int a = 10;           // 栈上，函数返回时自动释放
    int* b = malloc(10);  // 堆上，需要手动free
    free(b);
}
```

## 面试要点

1. **内存层级**：寄存器→缓存→内存→存储的速度容量权衡
2. **虚拟内存**：作用、分页、页面置换算法
3. **栈 vs 堆**：管理方式、速度、适用场景
4. **HDD vs SSD**：原理差异、性能差异
5. **内存泄漏**：堆内存未释放的后果，如何避免
6. **缓存一致性**：多核CPU缓存与内存同步问题
7. **缓存映射**：直接映射、组相联、全相联的区别
8. **缓存写策略**：Write-Through vs Write-Back
9. **局部性原理**：时间局部性和空间局部性在程序中的应用
10. **缺页异常**：触发条件、操作系统处理流程
11. **页面置换算法**：LRU、FIFO、Clock算法
12. **内存对齐**：为什么需要字节对齐，对性能的影响
